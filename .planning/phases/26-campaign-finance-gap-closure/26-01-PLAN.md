  <what-built>
    Task 1 produced an audit listing every gap candidate. MATCH candidates will be auto-seeded by Task 3's `--fix --ingest` run. NO MATCH and AMBIGUOUS candidates will NOT be auto-seeded — they need human judgment.

    For each NO MATCH candidate, the script could not find any committee whose `cand_name` or `cmt_nm` contains the candidate's last name. This means one of:
    (a) The candidate filed under a PAC or committee name not containing their surname (rare but real)
    (b) The candidate has not opened a campaign committee (legitimate no-filer — common for late-entry, withdrawn, or low-budget candidates)
    (c) The candidate's name in `essentials.politicians` differs from their Socrata `cand_name` (e.g., legal vs. preferred name)

    For each AMBIGUOUS candidate, multiple committees match the surname. This means one of:
    (a) Common surname collision (need first-name disambiguation, which the script already attempted and still found multiple)
    (b) The candidate has multiple committees (officeholder + campaign + ballot measure) — any one of them is correct
    (c) Surname coincides with an unrelated committee name
  </what-built>
  <how-to-verify>
    For each NO MATCH candidate from Task 1's output:

    1. Open https://ethics.lacity.org/disclosure/campaign/ and search by candidate name

       - Note the `cmt_id` (filer_id)
       - Note the committee name

    **Step 3c — Verify gap closure and contribution counts.**

    Run via psql:

    ```sql
    -- Query A: still_missing should be 0 (or equal to legitimate_no_filer + deferred count)
    SELECT COUNT(*) AS still_missing
    FROM essentials.race_candidates rc
    JOIN essentials.races r ON r.id = rc.race_id
    JOIN essentials.elections e ON e.id = r.election_id
    JOIN essentials.politicians p ON p.id = rc.politician_id
    WHERE e.state = 'CA'
      AND e.election_date >= CURRENT_DATE
      AND r.position_name ILIKE '%Los Angeles%'
      AND rc.candidate_status = 'active'
      AND NOT EXISTS(
        SELECT 1 FROM transparent_motivations.politician_sources ps
        WHERE ps.essentials_politician_id = p.id
          AND ps.source_system = 'la_socrata'
          AND ps.research_status = 'confirmed'
      );

    -- Query B: per-candidate source + contribution count
    SELECT
      p.full_name,
      r.position_name,
      ps.external_id AS cmt_id,
      ps.research_status,
      COUNT(c.id) AS contribution_count
    FROM essentials.race_candidates rc
    JOIN essentials.races r ON r.id = rc.race_id
    JOIN essentials.elections e ON e.id = r.election_id
    JOIN essentials.politicians p ON p.id = rc.politician_id
    LEFT JOIN transparent_motivations.politician_sources ps
      ON ps.essentials_politician_id = p.id
      AND ps.source_system = 'la_socrata'
      AND ps.research_status = 'confirmed'
    LEFT JOIN transparent_motivations.contributions c ON c.politician_source_id = ps.id
    WHERE e.state = 'CA'
      AND e.election_date >= CURRENT_DATE
      AND r.position_name ILIKE '%Los Angeles%'
      AND rc.candidate_status = 'active'
    GROUP BY p.full_name, r.position_name, ps.external_id, ps.research_status
    ORDER BY p.full_name;

    -- Query C: most recent la_socrata ingestion_runs row
    SELECT id, started_at, completed_at, records_fetched, records_inserted, status
    FROM transparent_motivations.ingestion_runs
    WHERE adapter_name = 'la_socrata'
    ORDER BY started_at DESC
    LIMIT 5;
    ```

    - Query C: the most recent la_socrata run has `status` IN ('completed', 'completed_with_warning'); status='failed' means the ingest broke and Step 3b must be diagnosed
