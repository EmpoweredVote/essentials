# Phase 169 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** Dark-mode design-system foundation. Reconcile the app-wide dark tokens to the Figma
style guide (node `3957:563`) as a single source of truth in `src/index.css`, and apply the dark
treatment to the **Results/Representatives page main content + global top nav/header chrome**
(`Layout.jsx` + ev-ui `Header`). This is a **re-theme + token** phase: NO new layout, NO banners
(Phase 170), NO Elections page (Phase 172), NO light-mode change, NO tile shape/size change.

**Design source:** Figma `J9mfnUSnc2k6fUQDhw9L7h` node `3957:563`; reference screenshot at
`scratchpad/figma/essentials-design.png`. Token values below are authoritative (from CONTEXT.md
D-02/D-05/D-06/D-07; verified against the screenshot — GitHub-dark page, full-bleed banner header,
teal eyebrow label, low-contrast surfaces, hairline borders, no drop-shadows).

**Look summary:** GitHub-dark. Depth comes from surface contrast (`#0d1117` page → `#161b22`
surface) + 8%-white hairline borders — **no drop-shadows**. Card radius 14px; control/button radius 10px.

## Spacing Scale

Declared values (multiples of 4). The mockup is a standard 8-point layout; these are the tokens
the executor must use on Results — no off-scale literals.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, eyebrow-to-title gap |
| sm | 8px | Compact element spacing, FilterBar control gap, chip padding |
| md | 16px | Default element spacing, card inner padding, gap between tiles |
| lg | 24px | Section padding, gap between government-body sections |
| xl | 32px | Layout gaps, tier-section vertical rhythm |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

**Radius (distinct from spacing — differentiates surfaces in the mockup):**
- Card / tile / surface radius: **14px**
- Control / button / input / dropdown radius: **10px**

Exceptions:
- Touch targets: interactive controls (dropdowns, search input, hamburger, "Visit website" icon
  links) keep `min-height/min-width: 44px` — existing accessibility rule, do not regress.
- `PoliticianCard` 4:5 tile: shape and size are **PRESERVED unchanged** (re-color only) —
  its existing internal radius/padding is not altered by this phase (success criterion #4).

## Typography

Fonts: **Inter** for body/UI, **Manrope** for display + section/eyebrow labels (D-05/06/07).
Weights used: **Regular (400)** + **SemiBold (600)** dominant.
Exactly the four sizes from the mockup — reconcile into type tokens, no inline literals.

**Weight exception (deliberate, approved):** **Bold (700)** is used on **exactly one element — the
hero/location title — and nowhere else.** This exceeds the default 2-weight heuristic by one weight,
on purpose: it is faithful to the Figma style guide (node `3957:563`) and is a locked user decision
(CONTEXT.md **D-06**: "Hero title = Manrope Bold 30px"). The body/UI ramp remains strictly 400 + 600;
no other element may use 700. Accepted as a justified single-use exception, not a typography smell.

| Role | Size | Weight | Line Height | Font | Notes |
|------|------|--------|-------------|------|-------|
| Label / meta | 12px | 600 (SemiBold) | 16px (1.33) | Manrope | Eyebrow labels: `uppercase`, letter-spacing **1.2px**, color teal `#00c8d7` (the "YOUR LOCATION" style). Dominant small-text size. |
| Body / names | 14px | 400 / 600 | 20px (1.43) | Inter | Politician names, body copy, control text. SemiBold for names, Regular for secondary. |
| Body large | 16px | 400 / 600 | 1.5 | Inter | Standard body / emphasized list text where 14px is too small. |
| Hero / display | 30px | 700 (Bold) | 36px (1.2) | Manrope | Page/location hero title (e.g. "Bloomington, IN"). Letter-spacing **-0.75px**. |

**Legibility rule (PROJECT, non-negotiable):** never faint-gray-on-dark. Muted text bottoms out at
`#8b949e` on the `#0d1117`/`#161b22` surfaces (passes contrast); do not go lighter/fainter than that
for any readable text.

## Color

Dark-mode palette reconciled to Figma (D-02). These **replace** the current dark token *values*
(`ev-navy #020618`, `ev-navy-card #1a2235`, `ev-teal-light #59b0c4`, body text `#f3f4f6`/`#d1d5db`).
Light-mode values **must not change** (D-03) — do not touch light teal `--ev-teal #00657c`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d1117` | Page background, app shell, header chrome base (replaces `#020618`) |
| Secondary (30%) | `#161b22` | Cards, tiles, controls, dropdowns, sidebar/nav surfaces (replaces `#1a2235`) |
| Accent (10%) | `#00c8d7` | See reserved-for list below (replaces dark `#59b0c4`) |
| Alert / notice | `#ff5740` (`ev-coral`) | Kept as-is (D-04) — alerts/notices (e.g. unopposed / no-candidate states). No Figma equivalent. |

**Text colors (dark):**
- Primary text: `#e6edf3` (replaces `#f3f4f6`)
- Muted / secondary text: `#8b949e` (replaces `#d1d5db`)

**Borders / dividers (dark):**
- Hairline border: `rgba(255,255,255,0.08)` (already matches — keep; this is the primary depth cue)
- Slate borders/dividers (heavier): `#2d3748` (default), `#4a5568` (emphasized/active)

**Accent (`#00c8d7`) reserved for — explicit list (never "all interactive elements"):**
- Eyebrow / section labels (the "YOUR LOCATION" uppercase 12px Manrope label)
- Government-body / jurisdiction section header text (currently forced to `#59b0c4`)
- Active filter state in `FilterBar` (active dropdown border + text, Compass toggle on-state,
  search-input focus border, checkbox `accentColor`)
- Hyperlinks and link-hover within themed surfaces (contact links, "Visit website" icons)
- Active tab indicator in the tab strip
Hover variant for links/accents: lighten toward `#7ec8d8` (existing hover convention) — keep.

**Depth / elevation:** NO drop-shadows. Card vs. page separation comes only from the
`#0d1117` → `#161b22` surface contrast + the 8%-white hairline border (D specifics). Do not add
`box-shadow` to cards or the header.

## Copywriting Contract

This phase is a **re-theme** of an existing page — it introduces no new flows, CTAs, or destructive
actions. The contract below records the existing strings that remain on the themed surface so the
checker can verify they stay legible on dark (it does not author new copy).

| Element | Copy |
|---------|------|
| Primary CTA | None new this phase. Existing primary action is the address lookup (owned by Landing, out of scope). Results has no new CTA. |
| Empty state heading | Preserve existing Results empty/no-results copy (e.g. name-search yields nothing) — re-theme legibly, do not reword. |
| Empty state body | Preserve existing. (Banner/section empty states are Phase 170.) |
| Error state | Preserve existing Results error/load-failure copy — ensure dark-legible (primary text `#e6edf3`, never faint gray). |
| Destructive confirmation | None — no destructive actions on Results. |

Eyebrow/label copy convention (visual, not new wording): section labels render `UPPERCASE` via the
12px Manrope SemiBold teal label style (e.g. "YOUR LOCATION"); this is a treatment, not a copy change.

## Implementation Constraints (for planner / executor)

These translate the locked decisions + PROJECT memory into hard rules the checker can verify:

1. **Single source of truth (DARK-01 / success #2):** every color, type, and spacing value used on
   Results must trace to a token in `src/index.css` (`@theme` keys or CSS vars). No scattered inline
   color literals on the Results surface.
2. **ev-ui overrides require `!important` (success #3):** `GovernmentBodySection`, `SubGroupSection`,
   `PoliticianCard`, and the ev-ui `Header` render inline styles. Dark theming lives in essentials
   `src/index.css` as `.dark .ev-<component> { … !important }`. No ev-ui repo change (D-08).
3. **No light-mode regression (success #5 / D-03):** change only dark-specific token values; leave
   `:root` light vars (`--ev-bg-light`, `--ev-teal #00657c`, etc.) untouched. Existing light-mode
   WCAG fixes (the `html:not(.dark)` blocks at end of `index.css`) must remain.
4. **PoliticianCard preserved (success #4):** 4:5 tile shape and size unchanged; re-color only
   (`background #161b22`, hairline border, primary/muted text per palette).
5. **No faint-gray-on-dark (PROJECT rule):** muted text floor is `#8b949e`; never lighter for
   readable text.
6. **No drop-shadows:** depth from surface contrast + hairline borders only.
7. **FilterBar + tab strip = minimal re-theme only (D-09):** make dark-legible (correct surface,
   accent, no light bleed, no faint gray); do NOT do a full Figma restyle — Phase 170 reworks this
   area. Keep the 44px touch targets.
8. **Tailwind v4 build safety (PROJECT rule):** `@source not "../.planning"` / `"../**/*.md"`
   hardening in `index.css` must stay; do not introduce raw Windows backslash paths in committed files.
