<objective>
Seed Maryland's 130 race rows for the 2026 General Election via two migrations:

- **Migration 279** (statewide, 12 rows): Governor, AG, Comptroller, US Senate (Van Hollen), and 8 US House CDs — using a single WITH gen_elec + VALUES list with the office_ids confirmed in RESEARCH.md.
- **Migration 280** (legislative scaffold, 118 rows): 47 SLDU senate districts (sequential geo_ids 24001..24047, all seats=1) + 71 SLDL house districts (29 whole-district seats=3 + 42 sub-district variable seats) — generated by a PowerShell script with hardcoded data arrays.

This plan covers MD-ELECTIONS-01 (race rows portion). Both migrations reference the election rows created by plan 96-01.

**Critical research findings to honor:**
- D-01 (CONTEXT.md): one row per DISTRICT, not per seat. `UNIQUE (election_id, position_name, primary_party)` enforces this.
- ROADMAP says "198 total race rows" — wrong. Actual count is 130 (12 statewide + 47 senate + 71 SLDL). Document the deviation in the migration 280 header comment.
- MD SLDL geo_ids are NON-SEQUENTIAL (mix of `24NNN` whole-district and `24NNA`/`24NNB`/`24NNC` sub-district). The generator MUST use a hardcoded data array. RESEARCH.md and PATTERNS.md contain the complete 71-row table — copy it verbatim.
- MD SLDU geo_ids ARE sequential (24001..24047, all seats=1) — a simple `for` loop is acceptable for senate.
- districts.state casing: `'md'` (lowercase) for STATE_UPPER/STATE_LOWER lookups; statewide races use hardcoded office_ids (no state casing needed).
- LG Aruna Miller has NO race row (same ticket as Governor). Angela Alsobrooks has NO race row (Class 2, not up in 2026). State Treasurer has NO race row (General Assembly-appointed). Only Van Hollen (Class 3) US Senate.

Purpose: Seed all scaffold race rows so the discovery agent (armed in plan 96-03) and the elections-by-address API can return them.
Output: 2 SQL migrations + 1 PowerShell generator + 2 apply scripts, all applied to production.
</objective>
