<objective>
Generate and apply migration 183: 372 empty legislative race scaffold rows (35 ME Senate + 151 ME House districts × 2 elections each). Uses a PowerShell generator script to produce all INSERT blocks — writing 372 blocks by hand would be error-prone. Candidate seeding is explicitly NOT done here; the discovery pipeline handles all legislative candidates automatically.

Purpose: Provides race rows so the discovery cron can match found candidates to the correct race when it runs. Without these rows, candidates discovered by the cron have no race to attach to.
Output: generate_me_legislative_races.ps1 + 183_me_2026_legislative_races.sql (applied to live DB)
</objective>

  <action>
Write generate_me_legislative_races.ps1 that outputs 183_me_2026_legislative_races.sql.

The script generates one SQL DO block per district (186 districts). Each DO block inserts two race rows (primary + general) for that district.

**Generator structure:**

```powershell
$output = @()
$output += "-- Migration 183: ME 2026 Legislative Race Scaffolding"
$output += "-- Generated by generate_me_legislative_races.ps1"
$output += "-- 35 Senate districts × 2 elections + 151 House districts × 2 elections = 372 rows"
$output += ""

# --- SENATE DISTRICTS (1..35) ---
for ($n = 1; $n -le 35; $n++) {
    $geo_id = "23" + ($n.ToString().PadLeft(3, '0'))  # '23001' through '23035'
    $pos = "ME State Senate District $n"

    $block = @"
DO `$`$
DECLARE
  v_primary_id UUID;
  v_general_id UUID;
BEGIN
  SELECT id INTO v_primary_id FROM essentials.elections
  WHERE name = '2026 Maine State Primary' AND state = 'ME';
  SELECT id INTO v_general_id FROM essentials.elections
  WHERE name = '2026 Maine General Election' AND state = 'ME';

  INSERT INTO essentials.races (election_id, office_id, position_name, primary_party, seats)
  VALUES (v_primary_id,
    (SELECT o.id FROM essentials.offices o
     JOIN essentials.districts d ON d.id = o.district_id
     JOIN essentials.chambers ch ON ch.id = o.chamber_id
     WHERE d.geo_id = '$geo_id' AND d.district_type = 'STATE_UPPER' AND d.state = 'me'
       AND ch.name = 'Maine Senate' LIMIT 1),
    '$pos', NULL, 1)
  ON CONFLICT (election_id, position_name) WHERE primary_party IS NULL DO NOTHING;

  INSERT INTO essentials.races (election_id, office_id, position_name, primary_party, seats)
  VALUES (v_general_id,
    (SELECT o.id FROM essentials.offices o
     JOIN essentials.districts d ON d.id = o.district_id
     JOIN essentials.chambers ch ON ch.id = o.chamber_id
     WHERE d.geo_id = '$geo_id' AND d.district_type = 'STATE_UPPER' AND d.state = 'me'
       AND ch.name = 'Maine Senate' LIMIT 1),
    '$pos', NULL, 1)
  ON CONFLICT (election_id, position_name) WHERE primary_party IS NULL DO NOTHING;
END `$`$;
"@
    $output += $block
}

# --- HOUSE DISTRICTS (1..151) ---
for ($n = 1; $n -le 151; $n++) {
    $geo_id = "23" + ($n.ToString().PadLeft(3, '0'))  # '23001' through '23151'
    $pos = "ME State House District $n"

    $block = @"
DO `$`$
DECLARE
  v_primary_id UUID;
  v_general_id UUID;
BEGIN
  SELECT id INTO v_primary_id FROM essentials.elections
  WHERE name = '2026 Maine State Primary' AND state = 'ME';
  SELECT id INTO v_general_id FROM essentials.elections
  WHERE name = '2026 Maine General Election' AND state = 'ME';

  INSERT INTO essentials.races (election_id, office_id, position_name, primary_party, seats)
  VALUES (v_primary_id,
    (SELECT o.id FROM essentials.offices o
     JOIN essentials.districts d ON d.id = o.district_id
     JOIN essentials.chambers ch ON ch.id = o.chamber_id
     WHERE d.geo_id = '$geo_id' AND d.district_type = 'STATE_LOWER' AND d.state = 'me'
       AND ch.name = 'Maine House of Representatives' LIMIT 1),
    '$pos', NULL, 1)
  ON CONFLICT (election_id, position_name) WHERE primary_party IS NULL DO NOTHING;

  INSERT INTO essentials.races (election_id, office_id, position_name, primary_party, seats)
  VALUES (v_general_id,
    (SELECT o.id FROM essentials.offices o
     JOIN essentials.districts d ON d.id = o.district_id
     JOIN essentials.chambers ch ON ch.id = o.chamber_id
     WHERE d.geo_id = '$geo_id' AND d.district_type = 'STATE_LOWER' AND d.state = 'me'
       AND ch.name = 'Maine House of Representatives' LIMIT 1),
    '$pos', NULL, 1)
  ON CONFLICT (election_id, position_name) WHERE primary_party IS NULL DO NOTHING;
END `$`$;
"@
    $output += $block
}

$output | Out-File -FilePath "183_me_2026_legislative_races.sql" -Encoding UTF8
Write-Host "Generated 183_me_2026_legislative_races.sql"
```

**Important implementation notes:**
- The geo_id format for STATE_UPPER uses '23001'..'23035' (PadLeft 3) — matching Phase 52 pattern
- The geo_id format for STATE_LOWER uses '23001'..'23151' (PadLeft 3) — same format
- Senate uses d.district_type='STATE_UPPER' and ch.name='Maine Senate'
- House uses d.district_type='STATE_LOWER' and ch.name='Maine House of Representatives'
- Both use d.state='me' lowercase (critical — different from elections.state='ME' uppercase)
- NO election_method column — it does not exist on essentials.races
- primary_party=NULL for all rows (combined race model)
- The LIMIT 1 prevents errors if district has no matching office (returns NULL office_id, which is allowed on races)
- House D29 (geo_id='23029') uses same pattern — if no office found (vacant seat), office_id will be NULL, which is fine
- Use backtick-escaped dollar signs (`$`$) in PowerShell heredoc for the $$ delimiters
- Run the script from C:/EV-Accounts/backend/migrations/ directory so the output file lands there
