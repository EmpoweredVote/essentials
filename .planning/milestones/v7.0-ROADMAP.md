## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- STATE_EXEC `districts.state` must be uppercase postal code; STATE/COUNTY lowercase — casing determines routing query match
- STATE_EXEC `district_id` should be empty string for shared multi-position districts (corrected from 'Oregon (Statewide)' pattern)
- Pre-existing CA seed existed before Phase 59 — always pre-check before CA state-level migrations
- `politician_images.type` must be `'default'` — UI filters `.find(img => img.type === 'default')`, 'headshot' causes silent invisibility
- Discovery agent creates politician rows with external_id=NULL — use UPDATE, not INSERT, when seeding already-discovered politicians
- Name collision pre-check required before seeding city officials — Matt Mahan duplicate (SJ Mayor + CA Gov challenger) required manual merge

**Issues Resolved:**
- LA County Supervisor sub-district geofences (X0005) loaded; routing now live
- LAUSD board members attached to sub-district geofences (not whole-district)
- Sacramento headshots behind AEM/CQ5 CMS — curl+grep background-image workaround documented
- Fremont fremont.gov 403 — Node.js browser User-Agent + Referer header workaround
- Matt Mahan duplicate row merged (race_candidates re-pointed, stale row deleted)

**Issues Deferred:**
- lavote.gov election ID changes each cycle — requires manual update after June 3 primary for November general (id=9fd492a8)
- Additional CA city deep seeds (Oakland, Long Beach, Anaheim, etc.)
- CA campaign finance ingestion (FPPC Form 460)

**Technical Debt:**
- CA districts.mtfcc swapped: STATE_UPPER has G5220, STATE_LOWER has G5210 (inverse of TIGER codes) — pre-existing data quality issue, routing unaffected
- CA pre-existing rows with state='CA' (uppercase) vs new rows state='ca' (lowercase) — 3 pre-existing LA County rows differ
- Broken idempotency in migration 193 (NATIONAL_LOWER CD offices) — safe in production but re-run would insert duplicates
