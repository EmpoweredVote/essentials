Create a new React functional component at src/components/MiniCompass.jsx. The component is pure-presentational and synchronous — no hooks except `useMemo` for derived data.

**Props (exact contract):**
```jsx
MiniCompass({
  userAnswers,        // [{ topic_id, value }, ...] from CompassContext
  polAnswers,         // [{ topic_id, value }, ...] — null/undefined safe; pass null when not fetched
  selectedTopics,     // string[] from CompassContext
  scopedTopics,       // topic objects pre-filtered to the race's districtScope (parent provides)
  invertedSpokes,     // object from CompassContext — passed through to RadarChartCore
  localLensActive,    // boolean from CompassContext
  isDark,             // boolean theme flag
  size = 120,         // outer container px (the "design" size — see scaling note)
})
```

**Imports:**
```jsx
import { useMemo } from 'react';
import { RadarChartCore } from '@empoweredvote/ev-ui';
import { computeDisplaySpokes, buildAnswerMapByShortTitle } from '../lib/compass';

const MAX_SPOKES = 8;
const INNER_SVG_SIZE = 200;  // RadarChartCore size — CSS scales it down to `size` container
```

**Component body:**

1. **Compute spokes via useMemo** so the work doesn't run on every parent re-render:
   ```jsx
   const { hasEnoughSpokes, topicsFiltered, userData, polData, hasReplacedSpokes } = useMemo(() => {
     if (!polAnswers || !userAnswers || userAnswers.length === 0 || scopedTopics.length === 0) {
       return { hasEnoughSpokes: false, topicsFiltered: [], userData: {}, polData: {}, hasReplacedSpokes: false };
     }
     const result = computeDisplaySpokes({
       selectedTopics,
       userAnswers,
       polAnswers,
       scopedTopics,
       maxSpokes: MAX_SPOKES,
       localLensActive,
     });
     if (!result.hasEnoughSpokes) {
       return { hasEnoughSpokes: false, topicsFiltered: [], userData: {}, polData: {}, hasReplacedSpokes: false };
     }
     const allowedShorts = result.displayTopicIds
       .map((id) => scopedTopics.find((t) => String(t.id) === id)?.short_title)
       .filter(Boolean);
     const { topicsFiltered: tFiltered, answersByShort: pMap } = buildAnswerMapByShortTitle(
       scopedTopics, polAnswers, allowedShorts
     );
     const { answersByShort: uMap } = buildAnswerMapByShortTitle(
       scopedTopics, userAnswers, allowedShorts
     );
     return {
       hasEnoughSpokes: true,
       topicsFiltered: tFiltered,
       userData: uMap,
       polData: pMap,
       hasReplacedSpokes: Object.keys(result.replacedSpokes).length > 0,
     };
   }, [polAnswers, userAnswers, selectedTopics, scopedTopics, localLensActive]);
   ```

2. **Silent absence:** If `!hasEnoughSpokes`, return `null` directly (no error UI, no placeholder). This satisfies CONTEXT.md "Silently absent when <3 total bilateral answers exist".

3. **Container + label-suppressed RadarChartCore:**
   ```jsx
   return (
     <div
       style={{
         width: size,
         height: size,
         flexShrink: 0,
         display: 'flex',
         alignItems: 'center',
         justifyContent: 'center',
         opacity: hasReplacedSpokes && localLensActive ? 0.7 : 1,
         borderRadius: '50%',
         border: `1px solid ${isDark ? 'rgba(255,255,255,0.10)' : 'rgba(0,0,0,0.08)'}`,
         boxSizing: 'border-box',
         padding: 4,
       }}
       aria-label="Mini compass"
     >
       <div style={{ width: '100%', height: '100%', display: 'flex', alignItems: 'center', justifyContent: 'center' }}>
         <RadarChartCore
           topics={topicsFiltered}
           data={userData}
           compareData={polData}
           invertedSpokes={invertedSpokes || {}}
           replacedSpokes={{}}              // labels suppressed, prop is moot — pass {} to keep simple
           boldOriginalSpokes={false}
           onToggleInversion={() => {}}     // read-only mini
           onReplaceTopic={() => {}}        // read-only mini
           size={INNER_SVG_SIZE}            // 200 — internal SVG coord system
           labelFontSize={0}                // SUPPRESSES LABELS — research pitfall #3
           padding={10}
           labelOffset={0}
           tightFit={true}
           darkMode={!!isDark}
         />
       </div>
     </div>
   );
   ```

**Critical pitfall notes — DO NOT deviate:**
- `INNER_SVG_SIZE = 200` not `size=120`. RadarChartCore tooltip foreignObjects (190px) and hit-dots (r=14) don't scale with size prop; at 120 they dominate. Pass 200 then CSS-constrain the container to 120. Research pitfall #3.
- `labelFontSize={0}` is the only way to hide labels — there is no showLabels prop. Research pattern #1.
- `tightFit={true}` is REQUIRED — without it, vertical whitespace from horizontal-cluster spokes makes the mini look broken.
- Always look up stances via `politician_id`, never `candidate_id`. Parent (ElectionsView in Plan 34-03) handles that mapping; this component just receives `polAnswers`.
- `replacedSpokes` prop on RadarChartCore only styles labels — irrelevant when labels are hidden. The visual distinction for replacement spokes uses container opacity (0.7) only when localLensActive=true AND replacements exist. When Lens is OFF, replacements are normal user-selected fallbacks (no visual distinction needed).

