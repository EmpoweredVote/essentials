## Layout Contract

### Position

| Property | Value |
|----------|-------|
| Position | `absolute` |
| Anchor | `top: 16px; right: 16px` |
| Text alignment | right-aligned (both lines) |
| Applies to | all three tiers (city, state, federal) — identical position at every tier |
| Breakpoint variance | **none** — identical geometry at 120px (mobile) and 180px (desktop) bands, matching the title block's own fixed-size-regardless-of-breakpoint precedent (only the container's height/margin/padding classes vary by breakpoint, never this component's inline text sizes) |

**Corner clearance (D-12):** the `16px` inset from both the top and right edges satisfies "kept clear
of the very corner" — do not reduce below `12px` on either axis, and do not set it to `0`.

**Non-collision proof (mobile, the tighter case):**
- Title block occupies the bottom band from `y = 68px` to `y = 104px` in the 120px-tall container
  (`bottom:0` anchor + `paddingBottom:16px` + `lineHeight:36px` = 52px consumed from the bottom edge).
- Feature-icon row (Phase 187) occupies `y = 68px` to `y = 104px` as well, but is bottom-**right**
  while the title is bottom-**left** — no horizontal overlap by design.
- This phase's stat block must have its total rendered height (padding + two lines + inter-line gap)
  fit inside `y = 16px` to `y ≤ 65px` — a **49px budget** with a 3px safety margin before the
  title/icon band begins at `y = 68px`. The grid-compliant geometry below (4px padding-top + 13px
  label line-height + 4px inter-line gap + 22px number line-height + 4px padding-bottom = **47px
  total**) fits this budget with 2px of slack. Every value is a multiple of 4. Do not increase font
  sizes or padding without re-verifying this arithmetic.
- On desktop (180px band), the same fixed geometry leaves ~130px of clearance above the title band —
  no risk there; the mobile calculation above is the binding constraint.

### Scrim (D-11 — same treatment family as Phase 187's `FeatureIconChip`)

| Property | Value |
|----------|-------|
| Background | `rgba(13, 17, 23, 0.55)` — literal rgba (not a CSS var; matches `FeatureIconChip`'s hardcoded value verbatim, since `--color-ev-navy` is `#0d1117` and this codebase's precedent hardcodes the alpha-blended literal rather than `color-mix()`) |
| Backdrop filter | `blur(2px)` — identical value to `FeatureIconChip`, so the two scrims read as one visual system |
| Border radius | `10px` (rounded rectangle — distinct from the icon chips' `50%` circle, but same "rounded, semi-transparent, navy" family) |
| Padding | `4px 12px` (vertical 4px, horizontal 12px) — both values are multiples of 4; the horizontal 12px matches the existing `src/index.css:71` `padding: 8px 12px` precedent |
| Layout | flex column, `align-items: flex-end` (right-aligns both lines as a block), `gap: 4px` between the label and number lines |
| Sizing | intrinsic (fits content) — do not set a fixed width; right-alignment via the `top/right` absolute anchor handles positioning, not a fixed box width |

### Inner text stack (top to bottom)

1. **Label** — uppercase, e.g. `POPULATION`
2. **Number** — full grouped digits, e.g. `652,503`

Both lines **always render together** — never label-only or number-only (D-10). There is no
"collapsed" or "abbreviated" mobile variant.

## Typography

This phase adds exactly **2 new type roles** (Label, Number) with exactly **2 weights** (600, 700).
The pre-existing Title and Tooltip rows are listed for reference only — they are unchanged by this
phase and are not part of this phase's "2 weight" budget.

| Role | Size | Weight | Line Height | Letter Spacing | Color | Font |
|------|------|--------|-------------|-----------------|-------|------|
| **Label** (NEW — this phase) | 11px | 600 (semibold) | 13px | 1px, uppercase | `var(--color-ev-text-muted)` (#8b949e) | `var(--font-sans)` |
| **Number** (NEW — this phase) | 20px | 700 (bold) | 22px | 0 (normal) | `var(--color-ev-text-primary)` (#e6edf3) | `var(--font-sans)` |
| Title (existing, unchanged) | 30px | 700 | 36px | -0.75px | `var(--color-ev-text-primary)` | `var(--font-display)` |
| Tooltip (Phase 187, unchanged) | 14px | 400 | normal | 0 | `#EBEDEF` on `#2F3237` chip | `'Manrope', sans-serif` |

**Label text:** exact string `POPULATION` (uppercase, no trailing colon, no unit). For the federal
tier this reads `UNITED STATES` as the location title (existing behavior) with `POPULATION` /
`334,914,895` as the stat — the label itself does not repeat the location name.

**Number format:** `toLocaleString()` grouped digits (e.g. `652,503`). No abbreviation ("653K"), no
suffix ("residents"), no leading unit symbol.

**Why Inter (`--font-sans`) for both new lines, not Manrope (`--font-display`):** the title uses
Manrope because it is the component's one "display" heading; the stat is a compact data readout, and
Inter's tighter digit/comma rendering at small sizes reads more cleanly than Manrope at 11-20px. This
also keeps Manrope reserved exclusively for the location title, preserving a clear typographic
hierarchy (display heading vs. metadata).

## Data / Rendering Contract (interaction states)

The `stats` prop on `SectionBanner` changes from an inert flag to a real data object:

- **Shape:** `stats: { label: string, value: number } | null | undefined`
  - Example (city tier): `{ label: 'POPULATION', value: 652503 }`
  - Example (federal tier): `{ label: 'POPULATION', value: 334914895 }`
- **Render condition:** render the scrim block **only when** `stats?.value` is a positive finite
  number (`typeof stats.value === 'number' && stats.value > 0`). Treat `null`, `undefined`, `0`,
  `NaN`, and any non-numeric value identically — **omit the entire block**, matching the existing
  `featureIcons?.length > 0 &&` conditional idiom already used in this file for the icon row (do not
  render an empty/zero-value scrim under any circumstance — STAT-03).
- **Label source:** pass `label` through the `stats` prop rather than hardcoding `POPULATION` inside
  `SectionBanner` itself — this keeps the component forward-compatible with a future second stat
  (deferred scope) without a visual contract change; for **this phase**, the resolver always supplies
  exactly `'POPULATION'`.
- **All three tiers eligible:** city, state, and federal banners all render the stat block when
  `stats.value` resolves (D-04 state always resolves via FIPS map; D-05 city resolves via browse
  `geo_id` OR name+state index; D-06 federal always resolves to the national total — no omit case for
  federal specifically, but the *same* conditional check still applies uniformly).
