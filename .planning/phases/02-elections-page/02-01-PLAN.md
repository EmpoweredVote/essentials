  <action>
Create `src/pages/Elections.jsx` as a new page component. This is the primary deliverable. Follow these requirements precisely:

**Imports and setup:**
- Import `Layout` from `../components/Layout`
- Import `ElectionsView` from `../components/ElectionsView`
- Import `fetchElectionsByAddress` from `../lib/api` (do NOT import `saveMyLocation`)
- Import `useCompass` from `../contexts/CompassContext`
- Import `useNavigate` from `react-router-dom` (for candidate click navigation to `/candidate/:id`)
- Import `useState`, `useEffect` from `react`

**State variables:**
- `electionsData` (array | null) — null = not yet fetched; array = fetched (possibly empty)
- `fetchLoading` (boolean) — true during API call
- `fetchError` (string | null) — error message or null
- `locationLabel` (string | null) — the address/label used for the last successful fetch
- `addressInput` (string) — controlled input value
- `showChangeInput` (boolean) — whether the "change location" address input is visible for connected users

**Tier-aware auto-load on mount (ELEC-02, ELEC-04):**
- Use `useCompass()` to get `isLoggedIn`, `userJurisdiction`, `compassLoading`
- CRITICAL: Gate ALL tier detection on `!compassLoading`. While `compassLoading` is true, render only the page heading "Elections" inside `<Layout>` — no address input, no loading spinner, nothing that would flash
- When `compassLoading` resolves to false:
  - If `isLoggedIn && userJurisdiction` is non-null: auto-fetch using `fetchElectionsByAddress` with `[userJurisdiction.city, userJurisdiction.state].filter(Boolean).join(', ')`. Set `locationLabel` to this address string. Do NOT show the address input
  - If NOT logged in OR `userJurisdiction` is null: show the address input + county shortcuts (this covers both Inform users and Connected-without-jurisdiction)
- Use a `cancelled` flag pattern in the useEffect cleanup to prevent stale state updates (same pattern as Results.jsx lines 334-346)

**Address input (ELEC-03, ELEC-06):**
- Plain `<input type="text">` with placeholder `"Enter your address"`
- Search button next to the input — both Enter key (onKeyDown) and button click trigger the same `handleSearch` function
- During fetch: show a spinner inside the search button (animate-spin pattern from the codebase), disable the button
- After successful fetch: set `electionsData` to the elections array, set `locationLabel`, clear `showChangeInput`
- Error handling: inline `<p>` below the input:
  - If `fetchError` is truthy (geocoder error, network error): "Address not recognized. Please enter a full street address."
  - If `fetchError` is null but `electionsData` is an empty array (no elections found for valid address): "No elections found for this address."
  - Clear error on next search attempt

**County shortcuts (ELEC-05):**
- Two buttons: "Monroe County" and "LA County"
- Hard-coded addresses (same as Landing.jsx COVERAGE_AREAS):
  - Monroe County: `'100 W Kirkwood Ave, Bloomington, IN 47404'`
  - LA County: `'500 W Temple St, Los Angeles, CA 90012'`
- Clicking a shortcut calls `handleSearch(hardcodedAddress)` — immediately fetches, no pre-fill
- VISIBILITY RULE: Only show shortcuts when `electionsData === null` (no results loaded yet). Once results are showing (even empty array), shortcuts disappear
- Style as secondary buttons consistent with the codebase (ev-teal border, clean look). Use the pattern from Landing.jsx as a reference but adapt for the Elections page context (these are smaller, less prominent than Landing county cards)

**Change location for Connected users (ELEC-06):**
- When auto-loaded results are showing (connected user path), display above the results:
  `"Showing elections for [locationLabel]"` with a `"Change"` link/button
- Clicking "Change" sets `showChangeInput = true`, which reveals the address input above the existing results. Results remain visible
- Submitting a new address via this input replaces the results (new fetch) but does NOT call `saveMyLocation` — this is a page-only lookup, not an account update
- After fetching with the new address, hide the change input (`showChangeInput = false`)

**Results rendering:**
- Pass `electionsData` to `<ElectionsView elections={electionsData} loading={fetchLoading} onCandidateClick={(id) => navigate('/candidate/' + id)} />`
- ElectionsView already handles: null state (returns null), empty state (shows "No upcoming elections"), loading skeleton, and full rendering with seeded shuffle
- Do NOT duplicate any ElectionsView logic

**Page layout:**
- Wrap everything in `<Layout>` for the site header
- Page-level heading: `<h1>` with text "Elections" — always visible, above everything
- Below heading: either the address input section (for Inform/no-jurisdiction) OR the "Showing elections for..." label (for connected auto-load)
- Below that: ElectionsView results
- Use `max-w-3xl mx-auto` or similar container for centered content (match the general page width pattern in the codebase)
- Standard padding: `px-4 py-6` or similar

**Route registration in App.jsx:**
- Add `import Elections from "./pages/Elections";` alongside other page imports
- Add `<Route path="/elections" element={<Elections />} />` inside the `<Routes>` block, after the existing routes

**Anti-patterns to avoid:**
- Do NOT show address input before `compassLoading` resolves (causes flash for Connected users — violates ELEC-02)
- Do NOT call `saveMyLocation` anywhere on this page
- Do NOT duplicate ElectionsView rendering logic (shuffle, grouping, etc.)
- Do NOT use Google Places autocomplete (plain text input only — see project constraints)
- Do NOT show a full-page loading spinner — only spinner is on the search button during fetch
