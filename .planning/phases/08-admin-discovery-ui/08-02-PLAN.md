Endpoint 1: GET /discovery/jurisdictions
Returns array of jurisdiction rows for the dashboard table.

```typescript
router.get('/discovery/jurisdictions', requireAuth as any, requireAdmin as any, async (req, res) => {
  try {
    const result = await pool.query(`
      SELECT
        dj.id,
        dj.jurisdiction_name,
        dj.election_date,
        dj.state,
        last_run.status         AS last_run_status,
        last_run.started_at     AS last_run_started_at,
        last_run.candidates_new AS last_run_candidates_added,
        active_cands.cnt        AS total_active_candidates
      FROM essentials.discovery_jurisdictions dj
      LEFT JOIN LATERAL (
        SELECT status, started_at, candidates_new
        FROM essentials.discovery_runs
        WHERE discovery_jurisdiction_id = dj.id
        ORDER BY started_at DESC
        LIMIT 1
      ) last_run ON true
      LEFT JOIN LATERAL (
        SELECT COUNT(*)::int AS cnt
        FROM essentials.race_candidates rc
        JOIN essentials.races r ON r.id = rc.race_id
        JOIN essentials.elections e ON e.id = r.election_id
        WHERE e.election_date = dj.election_date AND e.state = dj.state
      ) active_cands ON true
      ORDER BY dj.election_date ASC, dj.jurisdiction_name ASC
    `);
    res.json({ jurisdictions: result.rows });
  } catch (err: any) {
    console.error('[discoveryDashboard] GET /discovery/jurisdictions failed', err);
    res.status(500).json({ error: 'Failed to load jurisdictions', detail: err.message });
  }
});
```

Notes:
- Last run columns are NULL if the jurisdiction has never been run — frontend treats `last_run_status === null` as "never_run".
- `total_active_candidates` is COUNT across ALL races for this `(election_date, state)` pairing (not just races created by this discovery jurisdiction), per the research join chain.

Endpoint 2: GET /discovery/runs
Returns paginated run history.

```typescript
router.get('/discovery/runs', requireAuth as any, requireAdmin as any, async (req, res) => {
  const querySchema = z.object({
    limit: z.coerce.number().int().min(1).max(100).default(25),
    offset: z.coerce.number().int().min(0).default(0),
    jurisdiction_id: z.string().uuid().optional(),
  });
  const parsed = querySchema.safeParse(req.query);
  if (!parsed.success) {
    return res.status(400).json({ error: 'Invalid query', detail: parsed.error.flatten() });
  }
  const { limit, offset, jurisdiction_id } = parsed.data;
  try {
    const params: any[] = [limit, offset];
    let where = '';
    if (jurisdiction_id) {
      params.push(jurisdiction_id);
      where = 'WHERE dr.discovery_jurisdiction_id = $3';
    }
    const result = await pool.query(`
      SELECT
        dr.id,
        dr.discovery_jurisdiction_id,
        dr.status,
        dr.started_at,
        dr.completed_at,
        dr.triggered_by,
        dr.candidates_found,
        dr.candidates_new,
        dr.candidates_withdrawn,
        dr.candidates_auto_upserted,
        dr.error_message,
        dj.jurisdiction_name
      FROM essentials.discovery_runs dr
      JOIN essentials.discovery_jurisdictions dj ON dj.id = dr.discovery_jurisdiction_id
      ${where}
      ORDER BY dr.started_at DESC
      LIMIT $1 OFFSET $2
    `, params);
    const countResult = await pool.query(
      `SELECT COUNT(*)::int AS total FROM essentials.discovery_runs dr ${jurisdiction_id ? 'WHERE dr.discovery_jurisdiction_id = $1' : ''}`,
      jurisdiction_id ? [jurisdiction_id] : []
    );
    res.json({ runs: result.rows, total: countResult.rows[0].total, limit, offset });
  } catch (err: any) {
    console.error('[discoveryDashboard] GET /discovery/runs failed', err);
    res.status(500).json({ error: 'Failed to load runs', detail: err.message });
  }
});
```

Notes:
- This endpoint REQUIRES the `candidates_auto_upserted` column added by Plan 08-01 (migration 083). Do not use COALESCE — if migration is missing, the SELECT must fail loudly so the gap is obvious.
- Server-side pagination is implemented even though research suggests client-side is fine at current scale; keeping server-side pagination future-proofs us and matches stagingQueue conventions.

Endpoint 3: GET /discovery/coverage
Returns per-jurisdiction race coverage stats.

```typescript
router.get('/discovery/coverage', requireAuth as any, requireAdmin as any, async (req, res) => {
  try {
    const result = await pool.query(`
      SELECT
        dj.id,
        dj.jurisdiction_name,
        dj.election_date,
        COUNT(r.id)::int AS total_races,
        COUNT(CASE WHEN rc_count.cnt > 0 THEN 1 END)::int AS races_with_candidates,
        COUNT(CASE WHEN rc_count.cnt = 0 OR rc_count.cnt IS NULL THEN 1 END)::int AS races_zero_candidates
      FROM essentials.discovery_jurisdictions dj
      JOIN essentials.elections e ON e.election_date = dj.election_date AND e.state = dj.state
      JOIN essentials.races r ON r.election_id = e.id
      LEFT JOIN (
        SELECT race_id, COUNT(*) AS cnt FROM essentials.race_candidates GROUP BY race_id
      ) rc_count ON rc_count.race_id = r.id
      GROUP BY dj.id, dj.jurisdiction_name, dj.election_date
      ORDER BY dj.election_date ASC, dj.jurisdiction_name ASC
    `);
    res.json({ coverage: result.rows });
  } catch (err: any) {
    console.error('[discoveryDashboard] GET /discovery/coverage failed', err);
    res.status(500).json({ error: 'Failed to load coverage', detail: err.message });
  }
});
```

Notes:
- The join chain uses `(election_date, state)` per research Pitfall 5. If multiple elections match (rare; LA County edge case), races aggregate together — which is the desired behavior.
- A jurisdiction with no matching elections will not appear (INNER JOIN). This is correct: if there are no elections on that date for that state, there is nothing to cover.
