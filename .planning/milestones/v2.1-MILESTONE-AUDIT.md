## E2E Flow Status

### Flow 1: Cron Sweep Auto-Upsert ✓ COMPLETE
```
discoverySweep.ts Sunday 02:00 UTC
→ acquireRunLock (409 if already held)
→ runDiscoverySweep()
→ for each jurisdiction (sequential)
→ runDiscoveryForJurisdiction({ autoUpsert: true, suppressRunEmail: true })
→ agent finds candidates
→ high-confidence (official/matched) + raceId not null
  → autoUpsertToRaceCandidates() → race_candidates
  → candidate_staging audit row (status='approved', reviewed_by='cron')
→ uncertain → candidate_staging (status='pending')
→ sweep-summary email (when outcomes non-empty)
→ releaseRunLock()
```
Status: **COMPLETE**

### Flow 2: Manual Trigger → Stage → Admin Review ✓ COMPLETE (partial — see Flow 3)
```
POST /discover/jurisdiction/:id (X-Admin-Token)
→ acquireRunLock (409 if held)
→ 202 Accepted
→ fire-and-forget: runDiscoveryForJurisdiction({ triggeredBy: 'on_demand' })
→ candidates → candidate_staging (status='pending')
→ per-run review email (when candidatesStaged > 0)
→ admin opens /admin/staging in browser
→ StagingQueue.jsx loads pending rows via GET /api/admin/discovery/staging
→ admin sees candidate with confidence badge, citation URL, race context
→ admin clicks Approve or Dismiss
```
Status: **COMPLETE through dismiss; Approve breaks before race_candidates (see Flow 3)**

### Flow 3: Manual Approve → race_candidates ✓ COMPLETE (fixed 0d89b91)
```
Admin clicks Approve in StagingQueue.jsx
→ adminApi.approveStagingCandidate(id)
→ POST /api/admin/discovery/staging/:id/approve (JWT)
→ stagingQueueAdmin.ts: UPDATE candidate_staging SET status='approved'
→ autoUpsertToRaceCandidates({ raceId, fullName, source: 'discovery_admin' })
→ race_candidates row inserted (or already_present)
→ 200 OK { upsertResult: 'inserted' | 'already_present' | 'skipped_*' }
← StagingQueue.jsx removes row from UI
✓ voter can see the candidate
```
Status: **COMPLETE**
