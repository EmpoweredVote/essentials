## Gaps Found (Blockers)

### GAP-1 (BLOCKER): politicians.is_appointed=false on appointed Portland officials (Phase 77)

**File:** `C:/EV-Accounts/backend/migrations/231_portland_officials.sql` lines 115, 151  
**Description:** Raymond C. Lee III (external_id=-690003, City Administrator) and Robert L. Taylor (external_id=-690004, City Attorney) are seeded with `is_appointed=false` in the `politicians` table, despite being explicitly appointed positions. Their `offices.is_appointed_position` IS correctly set to `true`.  
**Impact:** Any code path that reads `politicians.is_appointed` to filter or display appointed officials will silently misclassify these two as elected. The UI currently reads `offices.is_appointed_position`, so the issue is not visually surfaced today — but the data inconsistency is a latent bug.  
**Fix required:**
```sql
UPDATE essentials.politicians
SET is_appointed = true
WHERE external_id IN (-690003, -690004);
```
**Evidence:** 077-REVIEW.md CR-01 flagged this; no 077-REVIEW-FIX.md was created; Phase 77 VERIFICATION.md checked `offices.is_appointed_position` but not `politicians.is_appointed`.
