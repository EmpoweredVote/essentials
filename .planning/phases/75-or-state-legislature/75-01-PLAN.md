Model on `C:/EV-Accounts/backend/migrations/generate_me_senate.ps1` (loaded above). Hard constraints — every one is non-negotiable; violating any produces an invalid migration:

1. **Chamber name** is exactly `'Oregon Senate'` (verified Phase 73 migration 222 — see 73-01-SUMMARY.md). Use subquery, never hardcode UUID.
2. **government_id subquery**: `(SELECT id FROM essentials.governments WHERE name = 'State of Oregon')` — used inside chamber subquery to disambiguate from any other 'Oregon Senate' name collisions (defensive).
3. **State column for districts** is lowercase `'or'` (per 72-02-SUMMARY.md Gate 5: "or|STATE_UPPER|30"). NEVER write `'OR'` (uppercase) for STATE_UPPER lookups — that returns 0 rows. (Uppercase 'OR' applies ONLY to STATE_EXEC per PROJECT.md Phase 74 lesson.)
4. **representing_state** is uppercase `'OR'` on the offices table (per Phase 74 OR pattern — matches Phase 51 ME executive pattern).
5. **Office title** is the bare string `'Senator'` — NO compound like `'Senator, District N'`. (Matches Phase 52 ME and Phase 61 CA — district label is reached via district_id FK.)
6. **GEOID format**: `'41' + lpad(district_num, 3, '0')`. PowerShell: `$gid = '41' + ([int]$r.dist).ToString().PadLeft(3, '0')`. SD-01 → '41001', SD-17 → '41017', SD-30 → '41030'.
7. **district_type='STATE_UPPER'** MUST be in WHERE clause to disambiguate from STATE_LOWER — SD-17 (geo_id='41017') and HD-17 (geo_id='41017') share geo_id values (per 75-RESEARCH.md Pitfall 1).
8. **External IDs** per D-10: SD-N → external_id = `-4110000 - N`. So SD-01 → -4110001, SD-17 → -4110017, SD-30 → -4110030.
9. **Politician flags**: `is_active=true, is_appointed=false, is_vacant=false, is_incumbent=true` on every politician row. Senators are voter-elected.
10. **Office flags**: `is_appointed_position=false, is_vacant=false` on every office row.
11. **Idempotency**: `ON CONFLICT (external_id) DO NOTHING` on politicians INSERT; `WHERE NOT EXISTS (SELECT 1 FROM essentials.offices o WHERE o.district_id = d.id AND o.chamber_id = ...)` on offices INSERT. (Matches Phase 52 ME pattern.)
12. **NO `slug` column** in any INSERT — slug is GENERATED ALWAYS on chambers/governments. Slug never appears in this migration (no chamber INSERTs at all here).
13. **NO `ALTER TABLE`, NO chamber INSERT, NO districts INSERT, NO election_races INSERT.** Structural-seeding only.
14. **NO `role_canonical` column** in politicians INSERT — let DB default handle it (matches Phase 52 ME pattern).
15. **Wrap full output** in `BEGIN; ... COMMIT;` once at file top/bottom, not per CTE block.
16. **office_id back-fill** at the end, scoped to `external_id BETWEEN -4110030 AND -4110001`, guarded by `p.office_id IS NULL`.
17. **UTF-8 without BOM** when writing the .sql file: use `[System.Text.UTF8Encoding] $false` (psql cannot parse UTF-8 BOM).

**Per-senator CTE block shape — emit exactly this for each of the 30 entries:**

```sql
-- ===== SD-{N} ({gid}): {full} ({party}) =====
WITH ins_p AS (
  INSERT INTO essentials.politicians
    (id, full_name, first_name, last_name, party, is_active, is_appointed,
     is_vacant, is_incumbent, external_id)
  VALUES (gen_random_uuid(), '{full_escaped}', '{first_escaped}', '{last_escaped}', '{party}',
          true, false, false, true, {ext_id})
  ON CONFLICT (external_id) DO NOTHING
  RETURNING id
)
INSERT INTO essentials.offices
  (id, district_id, chamber_id, politician_id, title, representing_state,
   is_appointed_position, is_vacant)
SELECT gen_random_uuid(),
       d.id,
       (SELECT id FROM essentials.chambers
        WHERE name = 'Oregon Senate'
          AND government_id = (SELECT id FROM essentials.governments WHERE name = 'State of Oregon')),
       p.id,
       'Senator', 'OR', false, false
FROM essentials.districts d
CROSS JOIN ins_p p
WHERE d.geo_id = '{gid}' AND d.district_type = 'STATE_UPPER' AND d.state = 'or'
  AND NOT EXISTS (
    SELECT 1 FROM essentials.offices o
    WHERE o.district_id = d.id
      AND o.chamber_id = (SELECT id FROM essentials.chambers
                           WHERE name = 'Oregon Senate'
                             AND government_id = (SELECT id FROM essentials.governments WHERE name = 'State of Oregon'))
  );
```

**office_id back-fill (single trailing block):**

```sql
UPDATE essentials.politicians p
SET office_id = o.id
FROM essentials.offices o
WHERE o.politician_id = p.id
  AND p.external_id BETWEEN -4110030 AND -4110001
  AND p.office_id IS NULL;
```
