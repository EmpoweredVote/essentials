## Tech Debt

### districtScope Filtering Gap for At-Large Local Offices

- **Severity:** Medium — UX/display issue; Cambridge Councillors see unscoped compass topics
- **Detail:** Backend `essentialsService.ts` line 474 returns `district_type: row.district_type ?? ''` (empty string when NULL). `Profile.jsx` derives `districtScope` from `pol.district_type` — empty string matches no branch, so `districtScope = null`. `CompassCard.jsx` line 44: `if (!districtScope || allTopics.length === 0) return allTopics` — null scope bypasses all filtering. At-large offices (no `district_id`) trigger this path. Cambridge City Council is entirely at-large.
- **Impact:** Phase 46 human verification confirmed compass renders correctly with local stances. The display shows more topics than intended but the data is correct. Non-local topics will simply show no stance for Cambridge councillors (expected UX).
- **Resolution options:**
  1. Profile.jsx fallback: if `!dt && pol.government_type === 'LOCAL'` → set `districtScope = 'local'`
  2. Backend fallback: return `district_type = 'LOCAL'` when office has no district_id but belongs to a LOCAL government
  3. At-large office schema: set a synthetic `district_id` on at-large Cambridge offices pointing to a LOCAL-typed district row
- **Files:** `C:/EV-Accounts/backend/src/lib/essentialsService.ts:474`, `C:/Transparent Motivations/essentials/src/pages/Profile.jsx:212-219`, `C:/Transparent Motivations/essentials/src/components/CompassCard.jsx:44`

### G4040 COUSUB Boundaries Not Loaded

- **Severity:** Medium (future city scope), Low (Cambridge scope)

- **Resolution:** Phase 48 (backlog) — extend load-state-tiger-boundaries.ts to include G4040 COUSUB layer for MA.
