Run authoritative SQL queries against the live database to confirm Phase 23 success criteria. The Focused Communities project shares the `inform` schema with the EV-Accounts backend, so the DATABASE_URL in `C:/EV-Accounts/backend/.env` is the canonical connection.

**Step 1 — Get the database connection string:**
```bash

```

Capture the URL (export it as `$DATABASE_URL` for the queries below). NOTE: STATE.md already records that supabase CLI v2.75.0 has no `db query` command — psql is required.

**Step 2 — Run the four verification queries.** Each query produces a single number; compare against the expected value.

**Query A — Topic count (expected: 10):**
```bash
psql "$DATABASE_URL" -t -c "
SELECT COUNT(*) FROM inform.compass_topics
WHERE topic_key IN (
  'residential-zoning', 'growth-and-development', 'public-safety-approach',
  'homelessness-response', 'economic-development', 'transportation-priorities',
  'local-environment', 'rent-regulation', 'local-immigration', 'city-sanitation'
)
AND is_live = true
AND went_live_at IS NOT NULL;
"
```
Expected: `10`. If lower, list which `topic_key`s are missing:
```bash
psql "$DATABASE_URL" -t -c "
SELECT k FROM (VALUES
  ('residential-zoning'), ('growth-and-development'), ('public-safety-approach'),
  ('homelessness-response'), ('economic-development'), ('transportation-priorities'),
  ('local-environment'), ('rent-regulation'), ('local-immigration'), ('city-sanitation')
) AS expected(k)
LEFT JOIN inform.compass_topics t ON t.topic_key = expected.k AND t.is_live = true
WHERE t.id IS NULL;
"
```

**Query B — Stance count (expected: 50):**
```bash
psql "$DATABASE_URL" -t -c "
SELECT COUNT(*) FROM inform.compass_stances s
JOIN inform.compass_topics t ON t.id = s.topic_id
WHERE t.topic_key IN (
  'residential-zoning', 'growth-and-development', 'public-safety-approach',
  'homelessness-response', 'economic-development', 'transportation-priorities',
  'local-environment', 'rent-regulation', 'local-immigration', 'city-sanitation'
)
AND t.is_live = true;
"
```
Expected: `50`. If lower, find which (topic_key, value) combinations are missing:
```bash
psql "$DATABASE_URL" -t -c "
WITH expected AS (
  SELECT k, v FROM (VALUES
    ('residential-zoning'), ('growth-and-development'), ('public-safety-approach'),
    ('homelessness-response'), ('economic-development'), ('transportation-priorities'),
    ('local-environment'), ('rent-regulation'), ('local-immigration'), ('city-sanitation')
  ) AS topics(k)
  CROSS JOIN (VALUES (1),(2),(3),(4),(5)) AS vals(v)
)
SELECT k, v FROM expected e
LEFT JOIN inform.compass_topics t ON t.topic_key = e.k AND t.is_live = true
LEFT JOIN inform.compass_stances s ON s.topic_id = t.id AND s.value = e.v
WHERE s.id IS NULL;
"
```

**Query C — Content completeness (expected: 50 rows with all 4 fields populated):**
```bash
psql "$DATABASE_URL" -t -c "
SELECT COUNT(*) FROM inform.compass_stances s
JOIN inform.compass_topics t ON t.id = s.topic_id
WHERE t.topic_key IN (
  'residential-zoning', 'growth-and-development', 'public-safety-approach',
  'homelessness-response', 'economic-development', 'transportation-priorities',
  'local-environment', 'rent-regulation', 'local-immigration', 'city-sanitation'
)
AND t.is_live = true
AND s.text IS NOT NULL AND length(trim(s.text)) > 0
AND s.description IS NOT NULL AND length(trim(s.description)) > 0
AND array_length(s.supporting_points, 1) >= 3
AND array_length(s.example_perspectives, 1) >= 3;
"
```
Expected: `50`. If lower, the content authoring did not produce all 4 fields for every stance.

**Query D — Scope role count (expected: 14, broken down as 10 LOCAL + 4 STATE):**
```bash
psql "$DATABASE_URL" -t -c "
SELECT role_scope, COUNT(*) FROM inform.compass_topic_roles r
JOIN inform.compass_topics t ON t.id = r.topic_id
WHERE t.topic_key IN (
  'residential-zoning', 'growth-and-development', 'public-safety-approach',
  'homelessness-response', 'economic-development', 'transportation-priorities',
  'local-environment', 'rent-regulation', 'local-immigration', 'city-sanitation'
)
AND t.is_live = true
AND r.is_required = true
GROUP BY role_scope ORDER BY role_scope;
"
```
Expected output:
```
 local | 10
 state | 4
```

**Query E — Verify the specific 4 multi-scope topics have STATE tags:**
```bash
psql "$DATABASE_URL" -t -c "
SELECT t.topic_key
FROM inform.compass_topics t
JOIN inform.compass_topic_roles r ON r.topic_id = t.id
WHERE t.topic_key IN ('growth-and-development', 'economic-development',
                      'transportation-priorities', 'rent-regulation')
AND r.role_scope = 'state'
AND t.is_live = true
ORDER BY t.topic_key;
"
```
Expected: 4 rows (`economic-development`, `growth-and-development`, `rent-regulation`, `transportation-priorities`).
