**Step 3 — Per-senator loop (iterate in external_id DESC order — SD-01 → SD-30, or DESC, pick one and stick):**

1. **Compute district_number** from geo_id: `district_num = int(geo_id[2:])` — '41017' → 17.

3. **Download the image to /tmp/or_senate/{politician_id}-raw.jpg**:

4. **PIL load and quality check**:
   ```python
   from PIL import Image
   img = Image.open('/tmp/or_senate/{politician_id}-raw.jpg').convert('RGB')
   w, h = img.size
   # Expected per RESEARCH.md: 115×130 for OR senators
   if w < 100 or h < 100:
       gap('image below usable resolution')
       continue
   ```
   Reject (and GAP) if image contains group shot, superimposed banner, or text-over-face. For 115×130 government portraits this is rare but enforce the quality rules.

5. **Crop to 4:5 THEN resize to 600×750 LANCZOS q90** (per D-05, D-06, RESEARCH.md Pattern 3):
   ```python
   w, h = img.size
   target_ratio = 4/5  # 0.8
   current_ratio = w / h
   if current_ratio > target_ratio:
       # too wide — center-crop width (typical for 115×130 source: 115/130 = 0.885)
       new_w = int(h * target_ratio)
       left = (w - new_w) // 2
       img = img.crop((left, 0, left + new_w, h))
   else:
       # too tall — top-crop (preserve face/eyes at 1/3 from top)
       new_h = int(w / target_ratio)
       img = img.crop((0, 0, w, new_h))
   img = img.resize((600, 750), Image.LANCZOS)
   img.save('/tmp/or_senate/{politician_id}-headshot.jpg', 'JPEG', quality=90)
   ```
   For the typical 115×130 source: crop becomes 104×130, then upscale 5.77× via LANCZOS to 600×750.
