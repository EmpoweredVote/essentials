MIGRATION STRUCTURE (model on `105_tx_congressional_house_officials.sql` and `107_*office_id_backfill.sql`):

```sql
-- Migration 109: TX State Senate officials (89th Legislature)
--
-- Seeds 30 TX state senators + 31 offices (D1..D31; D4 is vacant — office row
-- exists with is_vacant=true, no politician).
--
-- Depends on:
--   - Plan 21-01: 31 STATE_UPPER districts (geo_ids '48001'..'48031') exist
--   - Plan 21-02: 'Texas State Senate' chamber exists under government_id 8aea8ed7-...
--
-- external_id pattern: -100400 - district_number
--   D1 Bryan Hughes  = -100401
--   D31 Kevin Sparks = -100431
--   D4 vacancy       = -100404 (intentionally unused — no politician row)
--
-- Idempotent:
--   - politicians INSERTs use ON CONFLICT (external_id) DO NOTHING
--   - offices INSERTs guarded by NOT EXISTS on (district_id, chamber_id)
--   - office_id back-fill at the end uses `WHERE p.office_id IS NULL` guard

BEGIN;

-- ===== Per-district seeding (30 senators + 1 vacancy) =====

-- D1: Bryan Hughes (R)
WITH ins_p AS (
  INSERT INTO essentials.politicians
    (id, full_name, first_name, last_name, party, is_active, is_appointed,
     is_vacant, is_incumbent, external_id)
  VALUES (gen_random_uuid(), 'Bryan Hughes', 'Bryan', 'Hughes', 'Republican',
          true, false, false, true, -100401)
  ON CONFLICT (external_id) DO NOTHING
  RETURNING id
)
INSERT INTO essentials.offices
  (id, district_id, chamber_id, politician_id, title, representing_state,
   is_appointed_position, is_vacant)
SELECT gen_random_uuid(),
       d.id,
       (SELECT id FROM essentials.chambers
         WHERE name = 'Texas State Senate'
           AND government_id = '8aea8ed7-5abd-46f7-be0f-2bbbfe9fd2d9'),
       p.id,
       'Senator', 'TX', false, false
FROM essentials.districts d
CROSS JOIN ins_p p
WHERE d.geo_id = '48001' AND d.district_type = 'STATE_UPPER'
  AND NOT EXISTS (
    SELECT 1 FROM essentials.offices o
    WHERE o.district_id = d.id
      AND o.chamber_id = (SELECT id FROM essentials.chambers
                           WHERE name = 'Texas State Senate'
                             AND government_id = '8aea8ed7-5abd-46f7-be0f-2bbbfe9fd2d9')
  );

-- D2: Bob Hall (R)
-- ... [repeat the same pattern for D2 through D31, skipping D4]

-- D4: VACANT — office row with politician_id NULL, is_vacant=true
INSERT INTO essentials.offices
  (id, district_id, chamber_id, politician_id, title, representing_state,
   is_appointed_position, is_vacant)
SELECT gen_random_uuid(),
       d.id,
       (SELECT id FROM essentials.chambers
         WHERE name = 'Texas State Senate'
           AND government_id = '8aea8ed7-5abd-46f7-be0f-2bbbfe9fd2d9'),
       NULL,         -- no politician
       'Senator', 'TX', false, true   -- is_vacant = true
FROM essentials.districts d
WHERE d.geo_id = '48004' AND d.district_type = 'STATE_UPPER'
  AND NOT EXISTS (
    SELECT 1 FROM essentials.offices o
    WHERE o.district_id = d.id
      AND o.chamber_id = (SELECT id FROM essentials.chambers
                           WHERE name = 'Texas State Senate'
                             AND government_id = '8aea8ed7-5abd-46f7-be0f-2bbbfe9fd2d9')
  );

-- D5..D31: continue the pattern

-- ===== office_id back-fill =====
-- Pattern from migration 107: link politicians.office_id <- offices.id via offices.politician_id.
-- Scoped to the full senator external_id range -100431..-100401.

UPDATE essentials.politicians p
SET office_id = o.id
FROM essentials.offices o
WHERE o.politician_id = p.id
  AND p.external_id BETWEEN -100431 AND -100401
  AND p.office_id IS NULL;

COMMIT;
```

CRITICAL: write all 31 INSERT blocks (30 senator blocks + 1 vacancy block for D4). Do NOT take shortcuts — repeat the WITH...INSERT pattern verbatim 30 times for the seated senators.

geo_id formula: `'48' + LPAD(district_number, 3, '0')` → 5-character TIGER GEOID. Examples: D1=`48001`, D9=`48009`, D14=`48014`, D31=`48031`. (Note: senate maxes at 31, so the formula's correctness is not stressed here, but the formula MUST match the one used in Plan 21-04 — house — for consistency. Do not write `'480' + LPAD(N, 2, '0')` — that produces a 6-character string for N >= 100, which would not match TIGER's 5-char geo_ids. The senate plan uses the correct formula even though no senate district number triggers the bug.)
