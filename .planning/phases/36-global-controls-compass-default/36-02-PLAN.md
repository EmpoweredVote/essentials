<objective>
Switch Elections.jsx from "always-derived compassMode for calibrated users" to "stateful compassMode with localStorage-backed auto-enable + user toggle", and replace the inline sticky controls block with the new shared `<CompassControlsBar />` component from Plan 36-01.

Purpose: Today Elections.jsx has `const compassMode = (userAnswers?.length ?? 0) >= 3` — pure derivation. Calibrated users always see compass mode; they have no way to turn it off. This violates the Phase 36 requirement that compass mode is a user-controllable default rather than an always-on derivation. Adopt the localStorage null-check pattern already proven in Results.jsx so that (a) first-visit calibrated users auto-enable compass mode, (b) explicit user off preference is respected on subsequent visits, and (c) uncalibrated users see no change.

Output: Elections.jsx refactored to use stateful `compassMode`, exposes a compass toggle checkbox in the existing filter controls row, and renders `<CompassControlsBar />` instead of the inline duplicated markup.
</objective>
