**A. db-foundation.md — add 2 patterns**

1. After the "Council Structure Decision Tree" section, add a new section "Senator Office Uniqueness (state legislatures)":
   - Problem: NATIONAL_UPPER district has 2 senators sharing one district_id (e.g., Maine: Collins + King share `district_id` for ME NATIONAL_UPPER)
   - Solution: uniqueness key on `essentials.offices` for senator rows is `(district_id, politician_id)`, NOT `(district_id, chamber_id)`. Chamber_id uniqueness blocks the 2nd senator.
   - Maine example: ME federal officials Phase 51-02 used external_ids -230101 (Collins) and -230102 (King) on the same NATIONAL_UPPER district_id — verified in migration 170.

2. Add a new section "Legislature-Elected Offices (state-specific)":
   - Problem: In some states (Maine, Tennessee, Virginia), the AG, SoS, or Treasurer is elected BY the legislature, not by voters. These offices are on no ballot.
   - Solution: `is_appointed_position=true` on the office row, and NO election race rows ever created for these positions.
   - Maine example: Frey (AG), Bellows (SoS), Perry (Treasurer) — all `is_appointed_position=true`; zero race rows in `essentials.races` for these chambers (verified in Phase 51-01).
   - Check before migrating: research how the state's executive constitutional officers reach office. Wikipedia state government page is sufficient for this check.

**B. officials-seed.md — add 3 patterns**

1. New section "Multi-Tier City Seeding Strategy" near the top:
   - Tier 1: deep seed (incumbents + emails + headshots + addresses); use for the largest 1-3 cities
   - Tier 2: incumbents only (no headshots, no addresses); use for next 5-10 cities
   - Tier 3-4: skeletal offices with `politician_id=NULL` + entries in `[STATE]-GAPS.md`; use for remaining cities
   - The `[STATE]-GAPS.md` file makes coverage visible. Silent omissions create false "we cover this" impressions. Always document Tier 3-4 cities so the next person knows what is and isn't seeded.
   - Maine example: Phase 53 = Tier 1 (Portland 18 officials full); Phase 54 = Tiers 2-4 (Lewiston/Bangor/SouthPortland/Auburn/Biddeford incumbents + 18 skeletal cities; gaps listed in ME-GAPS.md).
   - Tier 2 city external_id prefix pattern: 5-digit per-city prefix (e.g., Lewiston=-23387xxxx, Bangor=-23027xxxx) — keeps backfills traceable to the source migration.

2. New section "PowerShell Bulk-Seed Generator Pattern":
   - For 100+ row migrations, hand-writing SQL invites errors. Use a PowerShell generator script.
   - CRITICAL encoding rule: use `[System.IO.File]::WriteAllLines($path, $lines, [System.Text.UTF8Encoding]::new($false))` — the `$false` disables BOM. `Out-File` and `Set-Content` produce BOM/UTF-16 that PostgreSQL rejects.
   - Maine example: Phase 55-02 PowerShell generator produced migration 184 (372 legislative race rows) — script lives at C:/EV-Accounts/backend/scripts/.

3. New section "essentials.offices and essentials.politicians schema gotchas":
   - `essentials.offices` has NO `seat_label` column and NO `is_active` column — use embedded title pattern: `'Council Member (Ward N)'`, `'Council Member (At-Large N)'`, `'Council Member (District N)'`.
   - `essentials.offices` has NO `email` column — individual emails stored on `politicians.email_addresses` (TEXT[] array) as `ARRAY['addr@domain']` in INSERT VALUES.
   - `essentials.districts` has NO `short_label` column — only `label`.
   - UPDATE pattern for adding incumbent to existing skeletal office: match on `(chamber_id, title)` to identify the office row.
