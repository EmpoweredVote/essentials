Append a new exported function `computeDisplaySpokes()` after the Local Lens preset section (after `loadLocalLensState`, near the bottom of the file). The function MUST be a pure function — no React hooks, no module-level state, no I/O.

Signature:
```js
export function computeDisplaySpokes({
  selectedTopics,       // string[] of topic UUIDs (may exceed MAX_SPOKES — must be capped)
  userAnswers,          // [{ topic_id, value }, ...]
  polAnswers,           // [{ topic_id, value }, ...] — null/undefined safe
  scopedTopics,         // topic objects already filtered by districtScope
  maxSpokes = 8,
  localLensActive = false,
})
```

Returns:
```js
{
  displayTopicIds: string[],                  // ordered topic UUIDs to render as spokes
  replacedSpokes: { [short_title]: boolean }, // map of substituted spokes (label-distinct)
  hasEnoughSpokes: boolean,                   // true iff displayTopicIds.length >= 3
}
```

Algorithm (port directly from CompassCard.jsx lines 97-173 — keep exact behavior):

1. If `!polAnswers || scopedTopics.length === 0`, return `{ displayTopicIds: [], replacedSpokes: {}, hasEnoughSpokes: false }`.

2. Build `polAnsweredSet` = Set of String(topic_id) from polAnswers where value > 0. Build `userAnsweredSet` = Set of String(topic_id) from userAnswers (any value). Both must coerce ids via String() for safety.

3. **Lens-aware preferred IDs:**
   - If `localLensActive === true` → `preferredIds = LOCAL_LENS_TOPICS.slice(0, maxSpokes)`
   - Else if `selectedTopics && selectedTopics.length > 0` → `preferredIds = selectedTopics.slice(0, maxSpokes)`  (MAX_SPOKES guard — DO NOT remove; the calibration bug can set all 36 topics)
   - Else → fall through to bilateral-overlap fallback (step 6)

4. Build `topicById = new Map(scopedTopics.map(t => [String(t.id), t]))` and `preferredSet = new Set(preferredIds.map(String))`.

5. Build `replacementPool` = scopedTopics filtered to: NOT in preferredSet AND user answered AND pol answered. Initialize `ri = 0`. For each `id` in `preferredIds`:
   - Skip if `!topicById.get(String(id))`
   - If both `userAnsweredSet.has(String(id))` AND `polAnsweredSet.has(String(id))` → push to `displayTopicIds`
   - Else if `ri < replacementPool.length` → `const sub = replacementPool[ri++]; displayTopicIds.push(String(sub.id)); replacedSpokes[sub.short_title] = true;`
   - Else → drop this spoke (no replacement available)

6. **Fallback when no selectedTopics (and not Lens):** `displayTopicIds = scopedTopics.filter(t => userAnsweredSet.has(String(t.id)) && polAnsweredSet.has(String(t.id))).map(t => String(t.id))`.

7. Cap `displayTopicIds` at `maxSpokes` (slice(0, maxSpokes)).

8. `hasEnoughSpokes = displayTopicIds.length >= 3`.

9. Return `{ displayTopicIds, replacedSpokes, hasEnoughSpokes }`.

**CRITICAL constraints — do not remove:**
- The `selectedTopics.slice(0, maxSpokes)` cap is a known-bug guard (calibration sets all 36 topics selected). Never remove.
- String coercion via `String(id)` is required — topic IDs are UUIDs but defensive coercion was kept in the original. Preserve.
- `polAnsweredSet` uses `value > 0` (not just `value`) — preserve exactly.

**Import note:** `LOCAL_LENS_TOPICS` is already in this file (line 375); reference it directly (no import needed).

   // Build chart data via shared spoke-selection algorithm
   let topicsFiltered = [];
   let polData = {};
   let userData = {};
   let replacedSpokes = {};
   let hasEnoughSpokes = false;

   if (!polLoading && polAnswers !== null && scopedTopics.length > 0 && hasUserCompass) {
     const result = computeDisplaySpokes({
       selectedTopics,
       userAnswers,
       polAnswers,
       scopedTopics,
       maxSpokes: MAX_SPOKES,
       localLensActive,
     });
     hasEnoughSpokes = result.hasEnoughSpokes;
     replacedSpokes = result.replacedSpokes;

     if (hasEnoughSpokes) {
       const allowedShorts = result.displayTopicIds
         .map((id) => scopedTopics.find((t) => String(t.id) === id)?.short_title)
         .filter(Boolean);

       const { topicsFiltered: polTopics, answersByShort: polMap } = buildAnswerMapByShortTitle(
         scopedTopics,
         polAnswers,
         allowedShorts
       );

       polData = polMap;

         scopedTopics,
         userAnswers,
         allowedShorts
