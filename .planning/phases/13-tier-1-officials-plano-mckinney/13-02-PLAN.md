Migration structure — one DO block per politician, all wrapped in a single transaction:

```sql
-- Migration 092: Seed McKinney incumbent politicians (post-June 2025 runoff)
-- Depends on migration 088 (McKinney government + chamber + 7 offices)
-- 7 incumbents: Mayor + At-Large 1, At-Large 2 + District 1-4

BEGIN;

DO $$
DECLARE
  v_office_id UUID;
  v_politician_id UUID;
BEGIN
  -- Mayor — Bill Cox (June 2025 runoff)
  SELECT o.id INTO v_office_id
  FROM essentials.offices o
  JOIN essentials.chambers ch ON ch.id = o.chamber_id
  JOIN essentials.governments g ON g.id = ch.government_id
  WHERE g.geo_id = '4845744' AND o.title = 'Mayor';

  IF v_office_id IS NULL THEN
    RAISE EXCEPTION 'McKinney Mayor office not found — migration 088 must run first';
  END IF;

  INSERT INTO essentials.politicians (
    first_name, last_name, preferred_name, full_name,
    party, party_short_name,
    is_active, is_incumbent, is_vacant, is_appointed,
    office_id,
    valid_from, valid_to, term_date_precision,
    email_addresses, urls,
    data_source
  ) VALUES (
    'Bill', 'Cox', 'Bill', 'Bill Cox',
    NULL, NULL,
    true, true, false, false,
    v_office_id,
    '2025-06-01', '2029-05-01', 'month',
    NULL,  -- CloudFlare-protected; no confirmed address
    ARRAY['https://www.mckinneytexas.org/1167/Council-Members#Mayor'],
    'mckinneytexas.org'
  ) RETURNING id INTO v_politician_id;

  UPDATE essentials.offices SET politician_id = v_politician_id WHERE id = v_office_id;
END $$;

-- Repeat one DO block per remaining incumbent (At-Large 1, At-Large 2, District 1, 2, 3, 4) ...

COMMIT;
```

CRITICAL pitfalls from research (do not repeat):
1. **ALWAYS update `essentials.offices.politician_id` after each politician INSERT** — the service layer JOINs both directions; missing back-link breaks the API.
2. **`party = NULL` (not the string 'nonpartisan')** — Texas municipal is nonpartisan; NULL is the schema convention.
3. **`valid_from` and `valid_to` are TEXT columns**, not DATE — insert as ISO strings like `'2025-06-01'`.
4. **`term_date_precision = 'month'`** — exact swearing-in dates aren't consistently published.
5. **`is_appointed = false` for ALL McKinney** — all 7 are elected (Cox + Lynch via June 2025 runoff still counts as elected).
6. **No `slug` column on essentials.politicians** — politicians table does NOT have a generated slug like chambers does. Verify by inspecting schema before assuming.
7. **`data_source = 'mckinneytexas.org'`** for all 7 rows.
8. **Email is NULL for any incumbent without a confirmed address** — do NOT scrape CloudFlare-protected addresses, do NOT use the `{first_initial}{last_name}@mckinneytexas.org` pattern as a placeholder. Only include emails confirmed in the staging file.
9. **Geré Feltus**: preserve the é character — write the SQL file in UTF-8.
10. **Use `RAISE EXCEPTION` if office lookup returns NULL** — protects against silent FK orphans if migration 088 is missing.
11. **Do NOT hardcode any UUIDs** — always look up office_id by `g.geo_id = '4845744'` + `o.title = '...'` exact match.
