**Step 1: Build the Tier 3-4 work list:**

```sql
SELECT p.id, p.full_name, p.urls, g.name AS city,
       CASE
         WHEN g.name IN ('Anna', 'Melissa', 'Princeton', 'Lucas', 'Lavon', 'Fairview', 'Van Alstyne', 'Farmersville') THEN 3
         ELSE 4
       END AS tier
FROM essentials.politicians p
JOIN essentials.offices o ON o.id = p.office_id
JOIN essentials.chambers ch ON ch.id = o.chamber_id
JOIN essentials.governments g ON g.id = ch.government_id
LEFT JOIN essentials.politician_images pi ON pi.politician_id = p.id
WHERE g.state = 'TX'
  AND g.name IN (
    'Anna', 'Melissa', 'Princeton', 'Lucas', 'Lavon', 'Fairview', 'Van Alstyne', 'Farmersville',
    'Parker', 'Saint Paul', 'Nevada', 'Weston', 'Lowry Crossing', 'Josephine', 'Blue Ridge'
  )
  AND p.is_active = true
  AND p.is_vacant = false
  AND pi.id IS NULL
  AND p.photo_origin_url IS NULL
ORDER BY tier, g.name, o.title;
```

Expected count: ~74 politicians (Tier 3 ~45, Tier 4 ~29).

**Step 2: Process politicians city-by-city using the skill.** Per-city expectation:

- **Anna (Tier 3, HIGH availability):** Bio URL stored in `politicians.urls` (e.g., `annatexas.gov/NNNN/Name`). Individual portraits expected. Fallback: Ballotpedia.
- **Melissa (Tier 3, MEDIUM):** Individual EID URLs (`cityofmelissa.com/Directory.aspx?EID=XX`). Some have photos. Fallback: Ballotpedia.
- **Princeton, Fairview, Farmersville (Tier 3, LOW):** Only general council page URL stored (no individual bios). Browse the council page for per-member portraits. Fallback: Ballotpedia.
- **Lucas, Lavon, Van Alstyne (Tier 3, VERY LOW):** No bio URLs found in Phase 15. Go directly to Ballotpedia. Expect frequent gaps.
