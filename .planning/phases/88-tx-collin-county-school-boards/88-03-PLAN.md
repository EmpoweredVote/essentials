In subGroupOrderScore() (around line 382), find this block:

  if (
    pols.length > 0 &&
    pols.every(p => p.district_type === 'LOCAL_EXEC') &&
    EXECUTIVE_KW.some(kw => lower.includes(kw) || titleLower.includes(kw))
  ) return 10;

Change it to:

  if (
    pols.length > 0 &&
    (pols.every(p => p.district_type === 'LOCAL_EXEC') ||
      (EXECUTIVE_KW.some(kw => lower.includes(kw) || titleLower.includes(kw)) &&
       pols.every(p => LOCAL_EXEC_TITLE_RE.test(p.office_title || ''))))
  ) return 10;
  if (EXECUTIVE_KW.some(kw => lower.includes(kw) || titleLower.includes(kw))) return 20;

Rationale: TX city mayors have district_type='LOCAL' (not 'LOCAL_EXEC') because no separate LOCAL_EXEC district was created in the Collin County TX migrations. The original condition required LOCAL_EXEC district_type, causing ties (score 20) between Mayor and Council sub-groups. With alphabetical tie-breaking, "Council" (C) sorted before "Mayor" (M). The relaxed condition grants score 10 (sorts first) when ALL politicians in the sub-group have a mayor/governor title — regardless of district_type — so TX mayors sort before council members without any DB migration.

LOCAL_EXEC_TITLE_RE is already defined at the top of the file as /\b(mayor|governor)\b/i. Do not redefine it.
