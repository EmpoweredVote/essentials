## Duplicate-Check Query (run before each v15.0 city)

Before writing stances for any LA city, run this query (substitute the city geo_id) to confirm no orphaned pre-district records exist alongside the geofenced ones:

```sql
-- Step 1: Who is in the geofenced government?
SELECT p.full_name, p.external_id, p.id,
       COUNT(DISTINCT pa.topic_id) AS stances,
       COUNT(DISTINCT pi.id) AS images
FROM essentials.politicians p
JOIN essentials.offices o ON o.politician_id = p.id
JOIN essentials.chambers ch ON ch.id = o.chamber_id
JOIN essentials.governments g ON g.id = ch.government_id
LEFT JOIN inform.politician_answers pa ON pa.politician_id = p.id
LEFT JOIN essentials.politician_images pi ON pi.politician_id = p.id
WHERE g.geo_id = '0606308'  -- swap per city
GROUP BY p.full_name, p.external_id, p.id
ORDER BY p.full_name;

-- Step 2: Search by name for any orphans not in the geofenced government
SELECT p.id, p.external_id, p.full_name,
       COUNT(DISTINCT pa.topic_id) AS stances
FROM essentials.politicians p
LEFT JOIN inform.politician_answers pa ON pa.politician_id = p.id
WHERE p.full_name IN ('John Mirisch', 'Lili Bosse', ...)  -- actual names
GROUP BY p.id, p.external_id, p.full_name
ORDER BY p.full_name, p.external_id;
```

If duplicates found: write a dedup migration (same pattern as 713_alhambra_dedup.sql) before writing stances.
