## Milestone Summary

**Key Decisions:**

- Copeville (GEOID 4816600) excluded — may be unincorporated CDP; documented in migration header; add in follow-up if confirmed incorporated
- `slug` is a GENERATED column on `essentials.chambers` — never include in INSERT statements (caught and fixed in migration 088; documented for all subsequent migrations)
- Discovery domain allowlist restricted to `{collincountytx.gov, co.collin.tx.us, <city-official-domain>}` — no news/third-party sources; `collincountyvotes.gov` never used (domain doesn't exist)
- Test city for DISC-03 switched from Plano to Allen — Plano holds odd-year elections (no May 2026 races); Allen confirmed active Mayor race
- McKinney email pattern: role-based `{role}@mckinneytexas.org` — avoids CloudFlare blocks that prevent email verification on many TX city sites
- TX congressional `geo_id` pattern: `STATEFP(2) + CD119FP(zero-padded 2 digits)` = 4-char string (e.g., '4803' for TX-3)
- G4020 county boundary intersection pattern established: to add any TX county's congressional reps, (a) load G4020 boundary, (b) add `browseCountyGeoId` to COVERAGE_AREAS entry — no other code changes needed
- TX state legislative boundaries: `G5210=Senate=STATE_UPPER`, `G5220=House=STATE_LOWER` — matches TIGER spec AND `essentialsService.ts` MTFCC handling
- `geofence_boundaries.state = FIPS '48'`; `districts.state = abbreviation 'TX'` — intentional established pattern
- Prosper is legally a Town — `Town of Prosper`; Fairview is legally a Town — `Town of Fairview`; Fairview uses "Seat N" naming not "Place N"
- Princeton has 8 seats (Mayor + Place 1-7) — confirmed from official research; plan initially said 7

**Issues Resolved:**

- Migration 088 slug bug: slug is generated column on chambers — INSERT statements must omit it; caught during migration apply and fixed before production
- `politicians.office_id` backfill for TX House reps: Phase 19 set `offices.politician_id` but missed the reverse link; fixed in Phase 17-04 via migration 106 before any user-facing impact
- Collin County domain corrected to `collincountytx.gov` (Phase 16 research confirmed `collincountyvotes.gov` does not exist)

**Issues Deferred:**

- `discovery_jurisdictions` rows have `election_date='2026-05-02'` (past); cron sweep skips past-election rows — needs `election_date` update before next TX election cycle
- Post-election winner updates pending: 7+ seats across Allen/Frisco/Murphy/Celina need is_incumbent flips after May 2, 2026 TX election results are certified
- NOT-YET-SEEDED stubs: 10 Tier 3 + 9 Tier 4 seats awaiting post-election certification in migrations 097/098

**Technical Debt Incurred:**

- 3 phases (17, 20, 21) did verification inline in SUMMARY files rather than creating separate VERIFICATION.md artifacts — data is present and complete; only the artifact is missing
- Murphy/Celina/Prosper have 0 compass stances — documented as "NO stance evidence found"; evidence-only ingestion rule correctly enforced
