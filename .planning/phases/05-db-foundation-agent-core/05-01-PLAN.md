**Header comment block (mirror style of 042 and 069):**

Explain that this migration adds the storage layer for v2.1 Claude candidate discovery — three tables: `discovery_jurisdictions` (registry of discoverable places), `discovery_runs` (one row per agent invocation with JSONB audit output), and `candidate_staging` (one row per discovered candidate or withdrawal diff, pending admin review). Explicitly note: `discovery_runs` MUST be created BEFORE `candidate_staging` because `candidate_staging.run_id` is a FK to it.

Explain why `discovery_jurisdictions.jurisdiction_geoid` is `text` rather than a UUID FK: there is no `essentials.jurisdictions` table in the current schema. `essentials.elections` has no jurisdiction FK either. The existing codebase keys jurisdictions by `jurisdiction_geoid text` (see adminService, roleService, stanceService). We follow the same convention to avoid forcing a premature FK relationship.

**Step 1 — `essentials.discovery_jurisdictions`:**

```sql
CREATE TABLE IF NOT EXISTS essentials.discovery_jurisdictions (
  id                  uuid        NOT NULL DEFAULT uuid_generate_v4(),
  jurisdiction_geoid  text        NOT NULL,
  jurisdiction_name   text        NOT NULL,
  state               character(2) NOT NULL,
  election_date       date        NOT NULL,
  source_url          text,
  allowed_domains     text[],
  created_at          timestamptz NOT NULL DEFAULT now(),
  updated_at          timestamptz NOT NULL DEFAULT now(),
  CONSTRAINT discovery_jurisdictions_pkey PRIMARY KEY (id)
);
```

Add a unique index enforcing that the same (jurisdiction_geoid, election_date) pair is only registered once:

```sql
CREATE UNIQUE INDEX IF NOT EXISTS idx_discovery_jurisdictions_geoid_date
  ON essentials.discovery_jurisdictions (jurisdiction_geoid, election_date);
```

**Step 2 — `essentials.discovery_runs` (must come BEFORE candidate_staging):**

```sql
CREATE TABLE IF NOT EXISTS essentials.discovery_runs (
  id                    uuid        NOT NULL DEFAULT uuid_generate_v4(),
  discovery_jurisdiction_id uuid    NOT NULL
                        REFERENCES essentials.discovery_jurisdictions(id) ON DELETE CASCADE,
  jurisdiction_geoid    text        NOT NULL,  -- denormalized for query ergonomics
  election_date         date        NOT NULL,
  status                text        NOT NULL DEFAULT 'running'
                        CHECK (status IN ('running','completed','failed')),
  started_at            timestamptz NOT NULL DEFAULT now(),
  completed_at          timestamptz,
  candidates_found      int         NOT NULL DEFAULT 0,
  candidates_new        int         NOT NULL DEFAULT 0,
  candidates_withdrawn  int         NOT NULL DEFAULT 0,
  error_message         text,
  raw_output            jsonb,
  triggered_by          text,      -- 'on_demand' | 'cron' | admin email from Phase 7
  CONSTRAINT discovery_runs_pkey PRIMARY KEY (id)
);

CREATE INDEX IF NOT EXISTS idx_discovery_runs_jurisdiction_started
  ON essentials.discovery_runs (discovery_jurisdiction_id, started_at DESC);

CREATE INDEX IF NOT EXISTS idx_discovery_runs_status
  ON essentials.discovery_runs (status);
```

**raw_output JSONB documentation comment:** Inline SQL comment documenting that `raw_output` holds `{ model, input_tokens, output_tokens, candidates: [{ full_name, citation_url, race_hint }, ...], errors?: [] }`. Exact shape is defined by discoveryService in Plan 05-03; this column is the audit trail for hallucination review.

**Step 3 — `essentials.candidate_staging`:**

```sql
CREATE TABLE IF NOT EXISTS essentials.candidate_staging (
  id                    uuid        NOT NULL DEFAULT uuid_generate_v4(),
  run_id                uuid        NOT NULL
                        REFERENCES essentials.discovery_runs(id) ON DELETE CASCADE,
  discovery_jurisdiction_id uuid    NOT NULL
                        REFERENCES essentials.discovery_jurisdictions(id) ON DELETE CASCADE,
  full_name             text        NOT NULL,
  normalized_name       text        NOT NULL,
  citation_url          text        NOT NULL,   -- NOT NULL enforces "no citation = no staging entry"
  race_hint             text        NOT NULL,
  race_id               uuid        REFERENCES essentials.races(id),                -- nullable
  matched_candidate_id  uuid        REFERENCES essentials.race_candidates(id),      -- nullable
  confidence            text        NOT NULL
                        CHECK (confidence IN ('official','matched','uncertain')),
  action                text        NOT NULL DEFAULT 'new'
                        CHECK (action IN ('new','withdrawal')),
  flagged               boolean     NOT NULL DEFAULT false,
  flag_reason           text,
  status                text        NOT NULL DEFAULT 'pending'
                        CHECK (status IN ('pending','approved','dismissed')),
  dismissed_reason      text,
  reviewed_at           timestamptz,
  reviewed_by           text,
  created_at            timestamptz NOT NULL DEFAULT now(),
  CONSTRAINT candidate_staging_pkey PRIMARY KEY (id)
);
```

Indexes on candidate_staging:

```sql
CREATE INDEX IF NOT EXISTS idx_candidate_staging_run_id
  ON essentials.candidate_staging (run_id);

CREATE INDEX IF NOT EXISTS idx_candidate_staging_status_pending
  ON essentials.candidate_staging (status)
  WHERE status = 'pending';

CREATE INDEX IF NOT EXISTS idx_candidate_staging_jurisdiction
  ON essentials.candidate_staging (discovery_jurisdiction_id);

CREATE INDEX IF NOT EXISTS idx_candidate_staging_flagged
  ON essentials.candidate_staging (flagged)
  WHERE flagged = true;
```

**Closing:** `COMMIT;`

**Avoid:**
- Do NOT use `CREATE INDEX CONCURRENTLY` (it cannot run inside a transaction block; this migration is wrapped in BEGIN/COMMIT for atomicity — the 069 exception does not apply here because we are creating brand-new empty tables).
- Do NOT add a UNIQUE constraint on `candidate_staging.citation_url` or `(run_id, full_name)` — the same candidate can legitimately appear across multiple runs, and even within a single run a name may be cited by multiple URLs.
- Do NOT add a NOT NULL on `raw_output` — rows are inserted with status='running' BEFORE the agent returns, so raw_output starts null.
- Do NOT create FK from `discovery_jurisdictions.jurisdiction_geoid` to any existing table. No such table exists; the geoid is referential by convention only (matches the pattern used in adminService/roleService/stanceService).
