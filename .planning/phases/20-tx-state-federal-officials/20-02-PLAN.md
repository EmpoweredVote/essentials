**Per-photo workflow (repeat 8 times):**

1. **Download** the Commons original to a temp path (e.g. `/tmp/{politician_id}-raw.jpg`). Use curl with a User-Agent header — Wikimedia rejects empty-UA requests:
   ```bash
   curl -A "EmpoweredVoteHeadshotImporter/1.0 (jadams@empowered.vote)" \
        -L -o "/tmp/{politician_id}-raw.jpg" \
        "{commons_image_url}"
   ```

2. **Headspace check** — open the raw image and confirm:
   - Eyes are at roughly 1/3 from the top (project memory: `feedback_headshot_cropping.md`).
   - Full head + shoulders are visible.
   - No additional people in the frame.
   - No campaign text/graphics over the face.
   - If the framing is wrong (e.g. tight crop where eyes are at the very top), reject and find an alternate Commons portrait of the same subject.

3. **Crop to 4:5 aspect, THEN resize to 600x750** (project memory: `feedback_headshot_resize_no_distort.md` — never stretch directly). Use PIL with LANCZOS resampling and quality=90:
   ```python
   from PIL import Image
   img = Image.open("/tmp/{politician_id}-raw.jpg")
   # Crop to 4:5 first (preserve face composition — eyes at ~1/3 from top)
   w, h = img.size
   target_ratio = 4/5  # width/height
   if w/h > target_ratio:
       # too wide — crop horizontally
       new_w = int(h * target_ratio)
       left = (w - new_w) // 2
       img = img.crop((left, 0, left + new_w, h))
   else:
       # too tall — crop vertically (preserve top for face)
       new_h = int(w / target_ratio)
       img = img.crop((0, 0, w, new_h))  # keep top portion
   # Resize to 600x750 with LANCZOS
   img = img.resize((600, 750), Image.LANCZOS)
   img.convert("RGB").save("/tmp/{politician_id}-headshot.jpg", "JPEG", quality=90)
   ```

4. **AskUserQuestion approval gate** — display the processed 600x750 image to the user and ask "Approve this headshot for {full_name}?" with options:
   - Approve — proceed to upload + DB insert
   - Try alternate Commons image — back to Task 1 for this subject only
   - Skip — record as a gap with reason
   This is the checkpoint enforced by `autonomous: false` on this plan. Per-photo approval is mandatory; no batch approval.

5. **Upload to Supabase Storage** (only after approval):

   curl -X POST \

     -H "Content-Type: image/jpeg" \
