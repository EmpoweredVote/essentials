  <action>
    Import the floating-ui primitives (useFloating, useHover, useFocus, useDismiss, useRole,
    useInteractions, FloatingPortal, offset, flip, shift, autoUpdate) from '@floating-ui/react' — already a
    dependency, no install. Because tooltip state must be per-lens, build the tooltip inside a small
    child component rendered once per lens (each floating-ui hook set owns one button), OR track an open
    key; the child-component approach mirrors IconOverlay's IconWithTooltip and is preferred.

    For each lens button:
    1. Add `aria-label={lens.name}` unconditionally (desktop AND mobile) so the accessible name survives
       when the visible label is hidden (Pitfall 1). Use `lens.description || lens.name` as the tooltip
       body text (the synthesized "Best Match"/custom lens has no description — this fallback matches the
       old title behavior; Open Question resolution).
    2. Remove the native `title={...}` attribute from the button (D-02: title fails keyboard/touch).
    3. Render the visible `<span>{lens.name}</span>` ONLY when `!isDesktop` (mobile keeps icon+label;
       desktop is icon-only). `renderLensIcon(lens)` renders in both modes — the gavel already maps to
       key 'judicial'.
    4. Wire the tooltip via useFloating (placement 'bottom', middleware [offset(8), flip(), shift({padding:8})],
       whileElementsMounted: autoUpdate). Gate it to desktop only: `useHover(context, { enabled: isDesktop })`,
       `useFocus(context, { enabled: isDesktop })`, and only render the FloatingPortal tooltip when
       `isDesktop`. Put `ref={refs.setReference}` directly on the existing `<button>` (do not wrap it in a
       span — it is already focusable).
    5. CRITICAL merge (Pitfall 2): the button already has manual onMouseEnter/onMouseLeave for the
       "Calibrate this lens?" prompt. Do NOT spread `{...getReferenceProps()}` bare next to them — pass the
       existing handlers through: `getReferenceProps({ onMouseEnter: existingEnter, onMouseLeave: existingLeave })`
       so floating-ui composes both. Verify the calibration prompt still appears on desktop hover.
    6. Suppress the new tooltip whenever the needs-calibration prompt is already showing for that lens
       (Pitfall 3): only render the tooltip when `!showPrompt`, so the two popups never stack.

    Preserve the OPAQUE chip surface for every state (chips overlay the banner photo) and the dark-mode
    `lightenForDark` handling — do not regress those. Style the tooltip box opaque and theme-aware
    (dark: #1f2937/#e5e7eb; light: #fff/#111827), zIndex above the chip row. Do NOT reproduce the removed
    native-title mechanism in a comment (comment-text discipline).
