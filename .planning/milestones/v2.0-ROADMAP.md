# Milestone v2.0: Elections Page

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 1-4
**Total Plans:** 4

## Overview

This milestone delivers a standalone Elections page at `/elections` — a dedicated, top-level civic destination that works for any user tier. Connected users with a stored location skip the address step entirely. All others get a clean address input with county shortcuts. The backend is fixed first so that races with zero candidates surface alongside contested and unopposed races.

## Phases

### Phase 1: Backend Fix

**Goal**: Races with zero candidates are returned by the backend, not silently dropped.
**Depends on**: None — backend is independent; can deploy before any frontend work.
**Plans**: 1 plan

Plans:

- [x] 01-01: Fix LEFT JOIN + null-safe dedup/grouping + shape test

**Details:**
LEFT JOIN on `essentials.race_candidates` in both Part A (geofence) and Part B (statewide) queries. Moved `candidate_status != 'withdrawn'` filter from WHERE clause to ON clause — keeping it in WHERE turns the LEFT JOIN back into an effective INNER JOIN (NULL != 'withdrawn' evaluates to NULL/falsy in PostgreSQL). Null-safe deduplication filter (skip Set operations for NULL candidate_id). Null-safe grouping loop (only push candidate when candidate_id is non-null). ElectionRow interface updated with nullable fields. Integration test shape assertion confirms `race.candidates` is always an array. All 9 tests pass.

### Phase 2: Elections Page

**Goal**: Users can reach a dedicated Elections page and see election results appropriate to their account state, without unnecessary address re-entry.
**Depends on**: Phase 1 (backend must return 0-candidate races so the page renders complete data).
**Plans**: 1 plan

Plans:

- [x] 02-01: Elections page with tier-aware auto-load, address input, shortcuts, change location

**Details:**
Created `src/pages/Elections.jsx` — standalone Elections page at `/elections`. Connected users with stored jurisdiction auto-load via `elections/me` endpoint (bypasses geocoding entirely, uses stored geo_ids directly — original plan used city+state geocoding which fails with Census Geocoder and returns wrong-district races). Inform and Connected-without-jurisdiction users see address input + Monroe County / LA County shortcuts. Change location flow reveals address input above existing results without calling `saveMyLocation`. `compassLoading` gate prevents address input flash for Connected users. Fixed hardcoded Indiana state labels ("Indiana General Assembly" / "Indiana Executive") in `ElectionsView.jsx` — replaced with "State Legislature" / "State Executive" for all states. Added `fetchMyElections()` to `src/lib/api.jsx`.

Files: `src/pages/Elections.jsx` (created), `src/App.jsx`, `src/lib/api.jsx`, `src/components/ElectionsView.jsx`

### Phase 3: Unopposed and Empty Race UX

**Goal**: Every race surfaces on the page with appropriate treatment — 1-candidate races are labeled unopposed, 0-candidate races show a notice, and nothing is hidden.
**Depends on**: Phase 2 (Elections page must exist); Phase 1 (0-candidate races must be present in API response).
**Plans**: 1 plan

Plans:

- [x] 03-01: Unopposed badge, empty notice, branch sort, and zebra-stripe in ElectionsView

**Details:**
Three-state race render guard derived from `activeCandidates.length` at render time. Contested races render normally. Unopposed races (1 candidate) show "Running Unopposed" absolute-positioned photo overlay (same pattern as Withdrawn banner). Empty races (0 candidates) show coral notice box using `pillars.empower.light` / `pillars.empower.textColor` design tokens. Branch-priority ordering within each government body (Executive → Legislative → Judicial). Local tier uses `bodyOrderScore` civic priority directly (Mayor → City Council → Other city → Township → School → County → Courts) — branch-first sort skipped for Local because BRANCH_ORDER breaks civic priority (Township Trustees sort before City Council). Left-border zebra stripe: `2px solid #E5E7EB` on odd-indexed races, `2px solid transparent` on even (preserves layout alignment). Background fill approach was invisible on all tier backgrounds.

Files: `src/components/ElectionsView.jsx`

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**

- Use `elections/me` for Connected auto-load (Census Geocoder unreliable with city+state; also returns wrong-district races for users not at the geocoded address)
- Elections page is view-only — never calls `saveMyLocation` (lookup only, not location-setting)
- "Running Unopposed" as photo overlay, not gray pill — SubGroupSection has no badge slot; overlay is consistent with Withdrawn banner pattern
- Left-border zebra stripe over background fill — `rgba(0,0,0,0.03)` invisible on all tier backgrounds; 2px `#E5E7EB` border is visible without conflicting with tier colors
- Local tier skips branch-first sort — `BRANCH_ORDER` correct for State/Federal but wrong for Local (Township Trustees are Executive branch, but should sort after City Council)
- navItems two-step computation in Layout.jsx (baseNavItems + spread) — clean separation of Read & Rank injection from Elections append; no mutation of `defaultNavItems`

**Issues Resolved:**

- 0-candidate races silently dropped by INNER JOIN — fixed with LEFT JOIN + ON-clause status filter
- Hardcoded Indiana state labels in ElectionsView — replaced with generic labels
- CA Governor and state/federal races missing from elections endpoint — accounts team fixed corrupt geofences (swapped MTFCC codes, county FIPS colliding with legislative district geo_ids)
- Census Geocoder returning wrong-district races for city+state input — resolved by switching to `elections/me`
- Stale local ev-ui dist (v0.1.53 vs npm v0.4.0) causing Vite build failure — synced local dist

**Issues Deferred:**

- CA Governor challenger candidates (10 filed, not yet seeded) — accounts team backlog
- LAUSD sub-district geofences (all 3 board races show for any LAUSD address) — accounts team backlog
- CA SoS challenger ingestion script — accounts team in progress
- Phase 2 missing VERIFICATION.md — documentation gap only; cross-validated by Phase 3 and 4 verifications

**Technical Debt Incurred:**

- None significant. All deviations in Phase 3 improved on the plan spec (overlay > pill, border > fill, civic priority > branch-first).
