## Phases

### Phase 22: Compass Schema Audit

**Goal**: Identify how scope filtering works in `inform.compass_stances` and determine whether "Criminalization of Homelessness" has enough politician answer data to warrant keeping vs. retiring
**Depends on**: Phase 21 (migration numbering baseline)
**Plans**: 1 plan

Plans:

- [x] 22-01: Document scope mechanism (compass_topic_roles), answer count (42), and retirement decision (keep both) in STATE.md

**Details:**

- `inform.compass_stances` has NO scope/level column — scope lives in `inform.compass_topic_roles` (join table with columns: topic_id UUID, role_scope TEXT, is_required BOOLEAN)
- Valid role_scope values: 'federal', 'state', 'local' — constrained by CHECK per migration 059
- `compassService.ts getCompassTopics()` converts compass_topic_roles rows to three booleans (applies_federal, applies_state, applies_local) at query time
- Topics with zero compass_topic_roles rows default to all-three-tiers (cross-cutting behavior)
- 42 politician answers for "Criminalization of Homelessness" topic — spans LA council, CA statewide, Indiana politicians; value distribution 1-5
- Retirement decision: KEEP BOTH — complementary framing (enforcement/criminalization vs. service delivery); 42 answers too substantial to orphan

## Milestone Summary

**Key Decisions:**

- Scope for compass topics lives in `inform.compass_topic_roles` join table — NOT in `inform.compass_stances` (no scope column there)
- Keep both "Criminalization of Homelessness" and "Homelessness Response" — complementary framing; 42 existing politician_answers too substantial to orphan
- 6 LOCAL-only + 4 LOCAL+STATE dual-scope topics — topics where state co-governs got dual scope
- local-immigration topic_key → immigration-policy slug — decouples public URL from internal key; avoids confusion with existing federal Immigration topic
- `t[key] !== false` filter in CompassCard.jsx — cross-cutting topics (undefined scope flags) correctly pass all tier filters
