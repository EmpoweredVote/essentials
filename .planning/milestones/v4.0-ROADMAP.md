## Milestone Summary

**Key Decisions:**

- computeDisplaySpokes() as single source of truth — extracted from CompassCard to compass.js; both CompassCard and MiniCompass share identical lens-aware spoke selection; never duplicate this algorithm
- INNER_SVG_SIZE=200 with CSS-constrained container — RadarChartCore foreignObjects (190px) and hit-dots (r=14) do not scale with size prop; always pass 200 internally, constrain via CSS wrapper
- Container opacity 0.7 ONLY when (hasReplacedSpokes && localLensActive) — replacement spokes only need visual distinction when Lens is forcing the topic set; when Lens OFF, replacements are normal user-selected fallbacks
- marginBottom: -70 load-bearing sticky overlay — must not be removed (scroll behavior for controls bar breaks)
- MINI-05+06 per-tile Lens icon superseded by CTRL-02 global controls bar — accepted design change 2026-05-14; global bar is the sole Local Lens entry point
- Phase 35 Hover Modal parked — spoke tooltips (Phase 34) + full-page compass navigation serve the information need; hover modal would conflict with tooltip layer
- Calibration threshold >= 3 (not >= 1) — kept at >= 3 to maintain parity between Elections.jsx and Results.jsx; changing both pages together is a product decision

**Issues Resolved:**

- CompassContext cross-app sync bug: ev-context authed slice captured as evCachedAnswers fallback when API returns empty (fix fca5adc)
- CompassContext SSO retry: 401 on /account/me now retries SSO before falling back to guest (fix fca5adc)
- Elections duplicate races: ElectionsView deduplicates races with identical candidate-ID sets before grouping (longer position name wins — fix c6e6fad)
- CompassKey dark mode: background + text + button colors applied with !important overrides (index.css)
- Elections race.candidates null crash: guarded + fetch .catch() handlers added (fix 3e6a319)

**Issues Deferred:**

- deriveScopedTopics defined identically in Results.jsx + ElectionsView.jsx — low priority, extract to src/lib/ in a future cleanup
- Phase 35 Hover Modal — parked; revisit only if a distinct use case emerges beyond spoke tooltips
- Elections.jsx Compass checkbox visible alongside auto-enabled compassMode — may confuse users who see compass already active (UX concern, not blocking)

**Technical Debt Incurred:**

- deriveScopedTopics duplication (Results.jsx line 35 + ElectionsView.jsx line 17) — identical function in two files
- No 34-VERIFICATION.md produced (human-verified interactively during session; all SUMMARY.md checkpoints passed)
- preLensSnapshot not exported from CompassContext value — future components cannot inspect "X topics saved" state
- enableCompass() called without compassLoading guard in Elections.jsx handleCompassModeChange — safe due to internal ref guard, but non-obvious behavior
