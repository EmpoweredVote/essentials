<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| district sites / Ballotpedia / Wikimedia (remote) → pipeline | Untrusted remote image bytes enter the pipeline over HTTP |
| pipeline → Storage/DB | Processed image + politician_images row cross into production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-201-IMG | Tampering | remote image fetch | mitigate | PIL `convert('RGB')` + JPEG re-encode `optimize=True` strips EXIF/embedded/stego payloads; only `image/jpeg` written to Storage |
| T-201-BIND | Spoofing | politician_images ↔ politician binding | mitigate | Fetch ONLY current-roster URLs; runtime UUID resolution by external_id prevents wrong-person binding on retry; Plan 06 human identity spot-check |
| T-201-LIC | Repudiation | image license provenance | mitigate | photo_license set per-image to the actual source license (never blanket us_government_work for a non-.gov source); recorded in the manifest |
| T-201-KEY | Information disclosure | SUPABASE_SERVICE_ROLE_KEY | mitigate | Key read only from gitignored `.env`; script itself gitignored (`backend/scripts/_*`); parameterized `external_id=%s` query (no injection) |
| T-201-WRONG | Spoofing | wrong-but-present image | transfer | Row-count cannot detect a wrong-but-present photo; deferred to Plan 06 human-verify spot-check |
| T-201-SC | Tampering | package installs | accept | No new pip installs — reuses installed Pillow/requests/psycopg2; Package Legitimacy Audit = N/A |
</threat_model>
