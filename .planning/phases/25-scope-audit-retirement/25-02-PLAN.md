    Add a `districtScope` prop to CompassCard (accepted values: 'local' | 'state' | 'federal' | null/undefined). When `districtScope` is provided, filter `allTopics` before passing to any downstream computation (chart building, accordion, etc.).

    Filtering logic — add this BEFORE the `if (compassLoading) return null` gate, using useMemo:

    ```jsx
    // Derive scope-filtered topics. If districtScope is not provided, use all topics (no filter).
    const scopedTopics = useMemo(() => {
      if (!districtScope || allTopics.length === 0) return allTopics;
      const key = districtScope === 'local'
        ? 'applies_local'
        : districtScope === 'state'
          ? 'applies_state'
          : 'applies_federal';
      return allTopics.filter((t) => t[key] !== false);
    }, [allTopics, districtScope]);
    ```

    Then replace every reference to `allTopics` INSIDE the component (after the gate) with `scopedTopics`. Specifically:
    - The `allowedShorts` derivation (where it iterates over allTopics to find answered topics)
    - The `allPolTopics` derivation
    - The `topicById` and `topicByShortLower` Maps in the chart-building block
    - Any place `allTopics` is passed as a prop to StanceAccordion or RadarChartCore

    The `useCompass()` destructure stays as `allTopics` — do not rename that. Only the local filtered variable changes.

    Note: `t[key] !== false` (not `t[key] === true`) because cross-cutting topics have `applies_local: true` and `applies_federal: true` and `applies_state: true`. Never use `=== true` which would exclude undefined; use `!== false` to treat undefined as true (safe default for cross-cutting).

    Do NOT touch the `useEffect` fetch for politician answers — that fetches all answers and is unrelated to scope display.
  </action>
