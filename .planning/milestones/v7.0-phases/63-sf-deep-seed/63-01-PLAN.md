**CRITICAL DIFFERENCE FROM ArcGIS LOADERS — DataSF is Socrata, not ArcGIS:**

- The URL is a plain `rows.geojson` endpoint. Do NOT append `?outSR=4326`, do NOT append `&f=geojson`, do NOT append `&where=1%3D1`. Those are ArcGIS-only parameters. Socrata returns native WGS84 GeoJSON.
- The district number field is `sup_dist_num` (numeric), NOT `DISTRICT` (ArcGIS string). Do not fall back to `DISTRICT` — that field does not exist on DataSF.

**Constants:**
```typescript
const DATASF_URL =
  'https://data.sfgov.org/api/views/f2zs-jevy/rows.geojson';

const MTFCC          = 'X0006';
const STATE          = '06';
const SOURCE         = 'sf_supervisor_districts_2022';
const EXPECTED_COUNT = 11;
const MAX_DISTRICT   = 11;
```

**Per-feature mapping:**
```typescript
const rawDistrict = props['sup_dist_num'];
const distNum = parseInt(String(rawDistrict ?? ''), 10);
if (isNaN(distNum) || distNum < 1 || distNum > MAX_DISTRICT) {
  console.warn(`  WARNING: sup_dist_num value '${rawDistrict}' out of range — skipping`);
  continue;
}
const geoId = `sf-supervisor-district-${distNum}`;
const name  = `District ${distNum}`;
```

Do NOT use `sup_name` (supervisor name) for the boundary `name` column — supervisor names change with elections; the district number is stable. The polygon `name` is just `District {N}`.

**INSERT SQL** (copy exactly from LA County supervisor loader, substituting the constants above):
```typescript
const result = await pool.query(`

    (geo_id, ocd_id, name, state, mtfcc, geometry, source, imported_at)

    ),
    $7, now())
  ON CONFLICT (geo_id, mtfcc) DO NOTHING
`, [geoId, geoId, name, STATE, MTFCC, geometryJson, SOURCE]);
```

Do NOT use `ST_MakeValid` — DataSF polygons are valid. The conflict key is `(geo_id, mtfcc)`.

async function querySfSupervisorBoundaries(
  client: Client, lon: number, lat: number
): Promise<Array<{ geo_id: string; name: string; mtfcc: string }>> {
  const res = await client.query(`
    SELECT geo_id, name, mtfcc
    FROM essentials.geofence_boundaries
