## BUG-01 — Systemic Cicero politician data corruption ~~(Backend — Critical)~~ **FIXED**

### Summary
The recent campaign finance data import corrupted the office→district chain for 50+ Cicero-sourced active politicians in California (and potentially other states). These politicians exist in the DB with `is_active: true` but have `district_type: null` and `geo_id: null` on their linked district, meaning they are invisible to the geofence-based address search.

### Confirmed broken politicians (CA, partial list — query was LIMIT 100)
Alex Padilla (US Senator), Andrea Alarcón, Andrew Gruel, Ardy Kassakhian, Bridgett Lewis, Butch Twining, Casey McKeon, Chad Williams, Cindy Allen, Claudia M. Frometa, Daniel Brotman, Daryl Supernaw, Debra Martin, Dionne Faulk, Don Kennedy, Dorothy Pemberton, Elen Asatryan, Eloy Morales, Eric Ohlsen, Gene Masuda, Gloria D. Gray, Gracey Van Der Mark, Hector Sosa, Jason Gibbs, Jason Lyon, Jennifer Perez, Jeremy Gerson, Jon Kaji, Joni Ricks-Oddie, Julia Ruedas, Justin Jones, Kristina Duggan, Laurene Weste, Letty Lopez-Viado, Lorraine Canales, Marvin Crist, Mary Zendejas, Megan Kerr, Nikki Perez, Ollie Cantos, Pat Burns, Patsy Ayala, Richard J. Loa, Rick Cole, Roberto Uranga, Sharon Kalani, Steve Madison, Suely Saro, Tamala Takahashi, Tony Wu, Vartan Gharpetian, Victor Preciado, Viviana Longoria

### Root cause (confirmed via audit — 2026-03-30)
Two independent issues from the campaign finance import:

**Issue A — District rows deleted:** The import deleted district rows that Cicero politicians' offices still reference. `offices.district_id` is set for all 54 affected politicians, but the corresponding `essentials.districts` rows no longer exist. This is **not** a null-field problem — a simple UPDATE won't fix it. District rows must be restored from backup or via a Cicero re-import for the affected politicians.

**Issue B — 76,332 CAL Access committee records in politicians table:** The `data_source: null` records are almost entirely `source = 'cal_access_discovery'` (76,332 of 77,634). These are campaign committees (PACs, officeholder accounts, etc.) and are not elected officials. The remaining ~1,302 null-`data_source` records (`ballotready`, `cicero`, `indiana_discovery`, `federal_*`, `scraped` sources) appear to be legitimate politicians with a missing field — do not quarantine these.

**Scope: CA only.** Indiana (13 active Cicero politicians) and federal/blank-state (25) are unaffected.

### Evidence
- Alex Padilla: `source: cicero`, `is_active: true`, but `offices.title = "Councilman"`, `districts.district_type = null`, `districts.geo_id = null`
- Padilla's original Senator office for CA (NATIONAL_UPPER) does not exist — only Adam Schiff's remains
- 77,626 rows with `data_source: null` match campaign committee naming patterns (e.g. "PADILLA OFFICEHOLDER COMMITTEE, ALEX", "PADILLA FOR SENATE", etc.)

### Impact
- All listed politicians are invisible to `POST /essentials/candidates/search` (geofence query returns no match)
- All listed politicians are invisible to `GET /essentials/representatives/me`
- Name search (NavSearch) returns them but clicking through shows no office/district data

### Required fix (backend team)
1. **Restore deleted district rows** for all 54 affected CA politicians — either from a pre-import database backup or by re-running the Cicero data import for these politicians. The `offices.district_id` FK values are intact; the district rows they reference are gone.
2. **Verify Alex Padilla's Senator office** specifically: confirm a `NATIONAL_UPPER` district for CA exists and is correctly linked (only Adam Schiff's currently remains; Padilla's was clobbered and now shows `title = "Councilman"`).
3. **Quarantine `cal_access_discovery` records** from `essentials.politicians` — the 76,332 rows with `source = 'cal_access_discovery'` are campaign committees, not elected officials. Move to a campaign finance entity table or mark with `is_active: false` / a dedicated flag. **Do not use `data_source IS NULL` as the filter** — ~1,302 legitimate politicians also have null `data_source`.

The full-state audit query (CA-only damage confirmed, but verify before closing):
```sql
SELECT p.full_name, p.source, o.title, o.representing_state, d.district_type, d.geo_id
FROM essentials.politicians p
JOIN essentials.offices o ON o.politician_id = p.id
LEFT JOIN essentials.districts d ON d.id = o.district_id
WHERE p.source = 'cicero'
  AND p.is_active = true
  AND (d.district_type IS NULL OR d.geo_id IS NULL)
ORDER BY o.representing_state, p.full_name;
```

> **Note:** The above query was used to confirm scope. The root cause is deleted district rows, not null fields — the JOIN returns null for `d.*` because no district row exists, not because the district row has null values.

### Resolution (2026-03-30 — live in production)
- **Issue A:** 43 deleted `essentials.districts` rows restored for all 54 affected CA Cicero politicians across 15 cities. Alex Padilla's `NATIONAL_UPPER` office for CA restored. Orphaned FK audit returns 0.
- **Issue B:** 76,332 `cal_access_discovery` committee records set to `is_active = false` and excluded from all search and geofence queries.

## BUG-02 — LocalUnavailableBanner hardcoded to Monroe County, Indiana ~~(Frontend — High)~~ **FIXED**

### Summary
Every politician profile that has no campaign finance data shows:

> "Campaign finance filings for this office are held by the **Monroe County Election Office** and are not yet available digitally."

with a mailto link to `election@co.monroe.in.us`.

This is shown for **all** politicians with no data — including California politicians, federal officials, etc.

### Location
`src/components/CampaignFinance/CampaignFinanceSection.jsx` — `LocalUnavailableBanner` component, lines 25–29.

### Root cause
The banner was written as a hardcoded placeholder for a specific Monroe County, Indiana test politician (appears during early local-politician development) and was never made generic or data-driven.

### Required fix (frontend)
The banner should show a generic message when no jurisdiction data is available, or ideally use the politician's actual `government_name` / `representing_state` to name the correct filing office.

Minimum fix — replace the hardcoded text with:

> "Campaign finance filings for this office may be held by a local or state election authority and are not yet available digitally."

And remove the `election@co.monroe.in.us` mailto link (or make it conditional on actually being a Monroe County politician).

### Resolution
Fixed in commit `dcd5fcb` (2026-03-30). Generic message now shown for all politicians without local filing data. Monroe County email removed.

### Original acceptance criteria (met)
- A Karen Bass profile page shows a generic unavailable message, not Monroe County ✓
- A Traci Park profile page shows a generic unavailable message, not Monroe County ✓
- The Monroe County email is not shown for any politician outside Monroe County, Indiana ✓

## BUG-03 — Prefilled login path missing city/city-council officials ~~(Backend — Medium)~~ **FIXED**

The `connected_profiles` table has no fields for city geo_id or city council district geo_id (`LOCAL`, `LOCAL_EXEC` district types). Quick-008 added fields for congressional, state senate, state house, county, and school district — but not city-level. Path 1 in `representatives/me` therefore cannot return Karen Bass (Mayor) or Traci Park (City Council) on first login. Manual address search works correctly.

See `ACCOUNTS-TEAM-NOTE-representatives-me-2026-03-30.md` for the full issue report sent to the Accounts team.

**Status:** Fixed (2026-03-30) — live in production. Path 1 now runs a hybrid query: geo_id lookup for the 5 pre-computed district types (congressional, state senate, state house, county, school district) plus a parallel coordinate-based query for `LOCAL`/`LOCAL_EXEC` types, merged before returning. No Census Geocoder call. Verified: user 4e6dde8f (12048 Culver Blvd, LA 90066) returns Karen Bass and Traci Park, ~17 results total.

Frontend workaround (`fix(prefilled):` commit) has been reverted — no action needed.
