# Phase 210.1 Plan 01: Fix Calibration Lens Revert (CR-01) Summary

**Seeded `tabLensMemory[activeView]` from the pending-calibration localStorage marker on
return-mount so a later async `rawUserAnswers`/`lenses` tick can no longer revert a
just-calibrated lens back to the tab default.**

## Performance

- **Duration:** ~20 min
- **Completed:** 2026-07-19
- **Tasks:** 1 completed
- **Files modified:** 1

## Accomplishments
- Closed Phase 210 code-review finding CR-01 (`.planning/phases/210-per-tab-compass-integration/210-REVIEW.md`):
  the calibrate-and-return race that could silently discard a user's just-completed lens
  calibration on a subsequent async `rawUserAnswers`/`lenses` update.
- Added a mount-time effect in `src/pages/Results.jsx` that reads `loadLensPending()` and, if a
  pending-calibration marker exists, seeds `tabLensMemory[activeView]` (raw URL param) with it —
  treating the calibration as the initiating tab's explicit pick, per Phase 210's D-04 pattern.
- Left the pre-existing tab-entry effect, `resolveTabLens`, `loadLensPending`/`saveLensPending`,
  and the `ev:compassLensPending` localStorage marker protocol completely untouched.

## Files Created/Modified
- `src/pages/Results.jsx` - Added `loadLensPending` to the compass import; added a new
  mount-once `useEffect` (empty dep array) directly above the existing tab-entry effect that
  seeds `tabLensMemory[activeView]` from `loadLensPending()` without clearing the marker.

## Decisions Made
- Keyed the seed on the raw `activeView` URL param rather than `effectiveActiveView`, per the
  plan's explicit interface note — `effectiveActiveView` isn't reliable yet on a fresh mount
  because it depends on `hasJudges`/`hasEducators`, which are still loading.
- Did not clear `ev:compassLensPending` in the new effect; ownership of clearing stays with
  `CompassContext.jsx`'s existing pending-lens adoption effect (lines 447-456), which runs after
  this child effect on the same commit per React's effect ordering.
- No changes to the tab-entry effect's dependency array or body — confirmed via `git diff`.
