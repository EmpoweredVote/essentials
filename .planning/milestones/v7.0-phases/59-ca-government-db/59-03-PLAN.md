**Per-photo workflow (repeat for each NEEDS_PHOTO official, in external_id order -06000101 → -06000108):**

1. **Download** the source image to a temp path:
   ```bash
   curl -A "EmpoweredVoteHeadshotImporter/1.0 (tjefferson@empowered.vote)" \
        -L -o "/tmp/{politician_id}-raw.jpg" \
        "{direct_image_url}"
   ```
   Include User-Agent header — Wikimedia rejects empty UA requests. If the URL is .png or .webp, use that extension for the temp file and PIL will handle the conversion.

2. **Headspace check** — open the raw image and confirm:
   - Eyes are at roughly 1/3 from the top
   - Full head + shoulders are visible
   - No additional people in the frame
   - No campaign text/graphics over the face
   If framing fails, reject and try the alternate source (swap official ↔ Wikipedia).

3. **Crop to 4:5 FIRST, then resize to 600x750** with PIL LANCZOS quality=90:
   ```python
   from PIL import Image
   img = Image.open("/tmp/{politician_id}-raw.jpg")
   img = img.convert("RGB")  # safe for webp/png input
   w, h = img.size
   target_ratio = 4/5  # width/height
   if w/h > target_ratio:
       # too wide — crop horizontally (center)
       new_w = int(h * target_ratio)
       left = (w - new_w) // 2
       img = img.crop((left, 0, left + new_w, h))
   else:
       # too tall — crop vertically (preserve top for face)
       new_h = int(w / target_ratio)
       img = img.crop((0, 0, w, new_h))
   img = img.resize((600, 750), Image.LANCZOS)
   img.save("/tmp/{politician_id}-headshot.jpg", "JPEG", quality=90)
   ```
   CRITICAL: crop 4:5 FIRST, then resize. Never call resize(600, 750) directly on the original — that stretches the face.

4. **AskUserQuestion approval gate** — display the processed 600x750 image and ask "Approve this headshot for {full_name} ({title})?" with options:
   - **Approve** — proceed to upload + DB insert
   - **Try alternate source** — swap official source ↔ Wikipedia for this subject
   - **Skip** — record as gap with reason

   Per-photo approval is mandatory; no batch approval allowed.

5. **Upload to Supabase Storage** (only after approval):
   ```bash
   curl -X POST \
     -H "Authorization: Bearer ${SUPABASE_SERVICE_ROLE_KEY}" \
     -H "Content-Type: image/jpeg" \
     -H "x-upsert: true" \
   ```

   Expected response: 200 OK with JSON `{"Key":"politician_photos/{politician_id}-headshot.jpg"}`.
