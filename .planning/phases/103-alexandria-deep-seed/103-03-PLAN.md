## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| alexandriava.gov / acps.k12.va.us -> Python script | Untrusted external image bytes downloaded; could contain malformed JPEG, EXIF metadata, or steganographic content |
| Python script -> Supabase Storage | Service-role key crosses into production storage; upsert=True allows overwrite |
| Migration 314 -> essentials.politician_images | URL strings hard-coded; could point to wrong bucket if pattern mistyped |
| Environment variable -> Python process | SUPABASE_SERVICE_ROLE_KEY in env; never logged or committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-103-03-01 | Tampering | Downloaded JPEG bytes from public websites | mitigate | Pillow re-encodes the image (decode + LANCZOS resize + JPEG encode at q90); strips EXIF and any stego payload; output is a freshly-generated JPEG |
| T-103-03-02 | Spoofing | Wrong bucket name 'politician-headshots' instead of 'politician_photos' | mitigate | Acceptance criteria forbids 'politician-headshots' string in script and migration; project memory explicitly notes this trap |
| T-103-03-03 | Information Disclosure | SUPABASE_SERVICE_ROLE_KEY leaked | mitigate | Read from env var only; never echo to logs or stdout; script does not write key to disk |
| T-103-03-04 | Tampering | Overlay text ("Re-Elect Smith") on candidate photo | mitigate | Manual visual check per feedback_headshot_no_graphics; skip-with-log policy in Task 1 acceptance criteria |
| T-103-03-05 | Denial of Service | All 7 Alexandria headshots fail to download -> migration EXCEPTION | mitigate | Task 1 escalation: if <7 SUCCESS for Alexandria, STOP before Task 2 and report — post-verification gate would otherwise block apply |
| T-103-03-06 | Tampering | Image distortion via direct resize without 4:5 crop | mitigate | Acceptance criteria requires LANCZOS resize AFTER 4:5 crop; project memory feedback_headshot_resize_no_distort documents the failure mode |
| T-103-03-07 | Tampering | URL row points to non-existent storage object | mitigate | Task 1 verification includes HEAD probe per uploaded URL; migration 314 INSERTs only after successful upload |
| T-103-03-SC | Tampering | npm/pip/cargo installs | mitigate | Script uses requests, Pillow, supabase-py — all existing project dependencies confirmed in prior phases (project_phase17_headshots references same stack); no new package installs introduced by this plan |
</threat_model>
