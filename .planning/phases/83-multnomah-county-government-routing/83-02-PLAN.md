<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public web → multco.us | Read-only fetch of public government photo data; no auth, no secrets |
| developer → Supabase Storage | Service-role write to politician_photos bucket via SUPABASE_SERVICE_ROLE_KEY |
| developer → Supabase Postgres | Service-role INSERT into essentials.politician_images |
| Supabase Storage → end users | Photos served as public objects (politician_photos bucket is public read) — by design |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-83-05 | Spoofing | multco.us source URL substitution | accept | URLs are from RESEARCH.md verified 2026-05-31; manual UI verification post-upload confirms correct face on each commissioner card (manual gate in VALIDATION.md). |
| T-83-06 | Tampering | image content modification | accept | Public-domain government portraits; even if multco.us were compromised, downstream impact is a display-only image, no PII. |
| T-83-07 | Information disclosure | SUPABASE_SERVICE_ROLE_KEY leakage | mitigate | Key loaded from local .env via dotenv; never logged; temp script deleted post-run (Task 2); never committed to git. |
