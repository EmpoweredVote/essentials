# Phase 170 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** Introduce a reusable `SectionBanner` component that divides the Results/Representatives
page into City → State → Federal in one continuous vertical scroll, remove the tier sort control,
and wire banner images + label copy to the extended `buildingImages.js` tier-image mapping. School
sections fold under the City banner (no 4th banner). Graceful fallback when no art exists: a dark
gradient band using the same eyebrow + pin + title. Stats data-slot and feature-icon slot exist as
hidden/empty scaffolding only. Requirements: **BANR-01, BANR-02, BANR-03, BANR-04, NAV-01**.

**Foundation locked by Phase 169:** All color tokens, typography, spacing scale, and ev-ui override
patterns were established in Phase 169 (`169-UI-SPEC.md`). This document **extends** that contract —
it does not re-derive it. Any value listed here that also appears in the 169 spec is a
direct carry-forward, not a new decision.

**Design source:** Figma `J9mfnUSnc2k6fUQDhw9L7h` node `3957:563`; reference screenshot at
`scratchpad/figma/essentials-design.png`. The mockup's skyline hero with `YOUR LOCATION` eyebrow +
`📍 Bloomington, IN` title is the direct visual precedent for every banner tier.

**Look summary:** Each banner = full-bleed dark band. Image banners: skyline/landmark photo with
dark gradient overlay from bottom (image still legible through the gradient). Fallback banners:
dark gradient band tinted by tier (no photo). Both carry the same eyebrow/pin/title layout. No
drop-shadows (GitHub-dark rule from 169 carries over).

## Spacing Scale

Carry-forward from Phase 169 (multiples of 4 only). No new tokens needed; the banner uses the
established scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Eyebrow-to-title gap; pin-to-title gap inside banner |
| sm | 8px | Compact inner padding; gap between eyebrow and lower content |
| md | 16px | Banner horizontal inner padding on mobile; gap between filter controls |
| lg | 24px | Banner horizontal inner padding desktop; spacing above first tier section |
| xl | 32px | Banner vertical padding (top/bottom) on desktop |
| 2xl | 48px | Vertical gap between the bottom of a banner and the first card row beneath it |
| 3xl | 64px | Page-level top/bottom spacing (unchanged from 169) |

**Banner dimensions (D-06):**
- Desktop height: **180px** (compact full-bleed band; not a hero)
- Mobile height: **120px**
- Width: **full bleed** via existing `-mx-6 md:-mx-12` + `px-6 md:px-12` pattern — edge-to-edge

**Radius:** Card / tile 14px; control / button 10px (carry-forward from 169; banners themselves are
flush full-bleed — no border-radius on the banner band itself).

Exceptions from Phase 169 carry forward:
- Touch targets: min 44px for interactive controls (preserved).
- `PoliticianCard` 4:5 tile: shape and size unchanged.

## Color

**Palette is carry-forward from Phase 169 (D-02).** Phase 170 adds only the fallback gradient
tints — no new base palette tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d1117` | Page background; banner fallback base tone |
| Secondary (30%) | `#161b22` | Cards, tiles, controls; banner gradient target at overlay center |
| Accent (10%) | `#00c8d7` | Banner eyebrow labels; active filter/tab states (carry-forward) |
| Alert / notice | `#ff5740` (`ev-coral`) | Coral pin SVG `noun-location-7814384-FF5740.svg` in banners; alerts/notices |
| Primary text | `#e6edf3` | Banner title text; primary card text |
| Muted text | `#8b949e` | Secondary card text (floor — never lighter on dark) |
| Hairline border | `rgba(255,255,255,0.08)` | Section dividers (no drop-shadows) |

**Accent (`#00c8d7`) reserved for — explicit list (carry-forward from 169, banner additions in bold):**
- **Banner eyebrow labels (`YOUR CITY` / `YOUR STATE` / `FEDERAL`)**
- Eyebrow / section labels (existing)
- Government-body / jurisdiction section header text
- Active filter state in `FilterBar`
- Hyperlinks and link-hover within themed surfaces
- Active tab indicator in the tab strip

**Banner gradient overlay (image banners):**
- Direction: `linear-gradient(to top, rgba(13,17,23,0.90) 0%, rgba(13,17,23,0.40) 50%, rgba(13,17,23,0.10) 100%)`
- Purpose: ensures title/eyebrow contrast; image still legible in upper half; no broken-image fallback needed
- Applied as a CSS `::after` pseudo-element (or `background` overlay layer) over the `<img>` tag

**Fallback gradient tints (D-10) — subtly per-tier, dark-theme tokens:**
- City tier: `linear-gradient(135deg, #0d1117 0%, #0d1822 100%)` — slight blue-navy lean
- State tier: `linear-gradient(135deg, #0d1117 0%, #121a17 100%)` — slight green-teal lean
- Federal tier: `linear-gradient(135deg, #0d1117 0%, #18120d 100%)` — slight amber-warm lean

These tints are deliberately subtle — they reinforce tier identity without competing with the dark
design system aesthetic. All three are readable at contrast minimums against `#e6edf3` title text.

**Depth rule (carry-forward):** NO drop-shadows. The banner gradient itself provides the depth cue;
all other depth comes from the `#0d1117` → `#161b22` surface contrast + hairline borders.

## Component Spec: `SectionBanner`

*Source of truth for the planner's component design. Executor implements to this contract.*

**File location:** `src/components/SectionBanner.jsx` (new essentials component).

**Props API (names at planner's discretion within these semantics):**

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `tier` | `'city' \| 'state' \| 'federal'` | yes | Determines eyebrow label, title, and fallback gradient |
| `locationName` | `string` | yes | The place name shown after the pin (e.g. `"Bloomington, IN"`, `"Indiana"`, `"United States"`) |
| `imageUrl` | `string \| null` | no | Image URL from `buildingImages.js`; when `null` or absent, renders the tier-tinted fallback gradient |
| `stats` | `object \| null` | no | Hidden/empty scaffolding slot — renders nothing when null (BANR-04). Structure TBD (planner's call). |
| `featureIcons` | `array \| null` | no | Hidden/empty scaffolding slot — renders nothing when null (BANR-04). Structure TBD (planner's call). |

**Eyebrow text (per tier, locked — D-02):**
- `city` → `YOUR CITY`
- `state` → `YOUR STATE`
- `federal` → `FEDERAL`

**Title text (per tier, computed from `locationName`):**
- All tiers → rendered as: coral pin SVG + space + `locationName`
- E.g. `📍 Bloomington, IN` / `📍 Indiana` / `📍 United States`

**Pin treatment (D-03):** Reuse existing SVG `public/images/noun-location-7814384-FF5740.svg`
as an `<img>` inline before the title text. Size: 20px × 20px (scaled from original). Color: coral
`#ff5740` — the SVG is already coral-colored, no `filter` needed. Do NOT recolor to teal.

**Full-bleed layout pattern (D-06, carry-forward from Results.jsx):**
- Outer wrapper: `-mx-6 md:-mx-12` (negative margin for edge-to-edge bleed)
- Inner content wrapper: `px-6 md:px-12` (re-establish padding for text content)
- Height: `h-[120px] md:h-[180px]`
- Position: `relative overflow-hidden` (for image + gradient overlay)

**Image rendering:**
- `<img>` tag: `object-fit: cover`, `width: 100%`, `height: 100%`, `position: absolute inset-0`
- Gradient overlay: CSS `::after` pseudo-element (or a `<div>` overlay sibling) on top of img,
  using the overlay gradient defined in Color section above
- Text content: `position: absolute bottom-0 left-0`, inside the padding container

**Fallback rendering (no `imageUrl`):**
- Replace `<img>` + overlay with the tier-tinted CSS `background` gradient from Color section
- Text content position identical to image variant

**Scaffolding slots (BANR-04 — hidden now, wired later):**
- `stats` slot: render `{props.stats && <div className="sr-only" />}` or equivalent — zero visual
  impact, DOM anchor exists for a later milestone to populate
- `featureIcons` slot: same — zero visual impact, DOM anchor exists

**School tier (D-07):** No `school` tier value in the prop. School sections render beneath the City
banner as normal sections; they do not trigger a new banner. The City `SectionBanner` must be placed
before both the Local AND School tier groups.

**Reuse target:** This component is also consumed in Phase 172 (Elections page parity, BANR-05).
Design the API so it works on both pages with no changes to the component signature.

## Banner Placement in Results.jsx

*For the planner — interaction between the new component and existing code.*

**Placement rule:** One `SectionBanner` before each top-level tier group:
- Before all `Local` + `School` sections → `<SectionBanner tier="city" ... />`
- Before all `State` sections → `<SectionBanner tier="state" ... />`
- Before all `Federal` sections → `<SectionBanner tier="federal" ... />`

**Tier sort control removal (D-11, NAV-01):**
- Remove the `Tier` / `TIER_OPTIONS` dropdown from `FilterBar.jsx`
- `selectedFilter` effectively becomes `'All'` permanently — simplify the dependent logic in
  `Results.jsx` (`filteredHierarchy` tier filter ~L1197, `locationLabel` ~L1218, the
  `selectedFilter === 'All'` eyebrow conditionals ~L1890/1914, empty-state gating ~L1882,
  `activeBuildingImage` branch ~L1232) — remove dead single-tier branches, keep all-tiers path

**Existing scroll-spy retirement (D-05):**
- Remove `activeBuildingImage` state + `IntersectionObserver` on `[data-tier]` (~L1232–1259)
- Remove the current swapping sidebar building image (`buildingImageMap` ~L1076)
- `data-tier` attributes may be kept as DOM anchors if useful; otherwise removed

**Existing section eyebrow removal (D-02):**
- Remove the per-section small eyebrow (`LOCAL` / `STATE` / `FEDERAL` at ~L1916) — the banner is now
  the single label per tier; no redundant double-labeling

**`FilterBar` after tier removal (D-11):** Elected/Appointed `Type` dropdown and name search are
preserved. Layout shift after removing the tier dropdown = planner's discretion (minimal reflow).

## Implementation Constraints (for planner / executor)

Carry-forwards from Phase 169 (all still apply) + Phase 170 additions:

1. **Single source of truth (DARK-01):** all color/type values used in `SectionBanner` trace to
   `src/index.css` `@theme` tokens. No new inline color literals on the banner.
2. **ev-ui overrides require `!important`:** applies to any ev-ui component visible inside or
   adjacent to banners. Banner itself is a first-party component — no `!important` needed for its
   own elements.
3. **No light-mode regression (D-03):** banner is dark-mode only (Results page runs dark). Do not
   introduce light-mode breakage. If `SectionBanner` is ever rendered outside `.dark`, it must not
   produce illegible white-on-white.
4. **PoliticianCard preserved:** shape and size unchanged. Banners appear between groups, not inside
   card grids.
5. **No faint-gray-on-dark (PROJECT rule):** banner eyebrow, title, and any text within the
   scaffolding slots must meet the `#8b949e` floor. Not applicable to the scaffolding slots (hidden).
6. **No drop-shadows:** the banner gradient is the only depth cue; no `box-shadow` on the banner.
7. **Gradient overlay is mandatory for image banners:** never raw image behind white text.
8. **School tier folds under City banner (D-07):** `SectionBanner` for `city` renders before Local
   AND School groups. Do not insert a 4th `school` banner.
9. **Stats + featureIcons slots render nothing when empty (BANR-04):** zero visual impact; the prop
   shape simply must exist for a later milestone to populate.
10. **Tailwind v4 build safety (PROJECT rule):** `@source not` hardening in `index.css` must stay;
    avoid raw Windows backslash paths in committed files.
11. **`SectionBanner` is reusable (success criterion #1):** one component renders all three tiers +
    image/gradient variants — do not fork per-tier components.
