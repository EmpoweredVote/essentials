SKIP CHECK — Run this query first to identify which assembly members already have headshots:
```sql
SELECT p.external_id, p.full_name, p.id as politician_id,
       pi.id as existing_image_id
FROM essentials.politicians p
LEFT JOIN essentials.politician_images pi ON pi.politician_id = p.id
WHERE p.external_id BETWEEN -6002080 AND -6002001
ORDER BY p.external_id DESC;
```
Note the politician_id for each assembly member and whether they already have a politician_images row. Skip upload (but still verify) for any that already have an image.

For each assembly district AD-01 through AD-80, process ONE AT A TIME:

DOWNLOAD URL PATTERN:
`https://webapi.assembly.ca.gov/district-media/assets/members/assembly_member_NN.jpg`
Where NN = zero-padded 2-digit district number (01, 02, ..., 80).

PROCESSING STEPS (per image):
1. Download assembly_member_NN.jpg — it is a 500x500 square JPEG
2. Crop center to 4:5 ratio BEFORE resizing — crop to 400x500 from the center:
   PIL: `img.crop((50, 0, 450, 500))` — removes 50px from left and right
   Do NOT resize directly from 500x500 to 600x750 — this distorts the image
3. Resize cropped 400x500 to 600x750 using Lanczos: `img.resize((600, 750), Image.LANCZOS)`
4. Save as JPEG quality=90

UPLOAD TO SUPABASE STORAGE:
- Bucket: politician_photos
- Path: {politician_id}-headshot.jpg (use the politician_id UUID from the skip check query)
- Get the public URL after upload

INSERT politician_images ROW (skip if politician already has an existing_image_id):
```sql
INSERT INTO essentials.politician_images
  (id, politician_id, url, type, photo_license)
VALUES
  (gen_random_uuid(),
   '{politician_id}',
   '{public_url_from_storage}',
   'default',
   'public_domain')
ON CONFLICT DO NOTHING;
```

HALT POLICY: If any assembly_member_NN.jpg URL returns 404 or error, HALT and flag which district is missing. Do not skip and continue.

Process assembly districts in order AD-01 through AD-80. After each successful upload + insert, proceed to the next district.

For each senator with no existing_image_id, source their headshot ONE AT A TIME using the /find-headshots skill.

PER-SENATOR URL APPROACH:
1. Primary URL: `https://sdNN.senate.ca.gov` (where NN = zero-padded 2-digit district)
   Example: SD-01 → https://sd01.senate.ca.gov, SD-10 → https://sd10.senate.ca.gov
2. If sd prefix returns 503 or 404, try `https://srNN.senate.ca.gov` (Republican senators)
   Example: SD-40 (Brian Jones) → https://sr40.senate.ca.gov
3. On the district page, look for:
   - `sdNN_headshot.jpg` or `SDxx_headshot.jpg` in the site files
   - A biography or about page with a full-size photo
   - Avoid footer/thumbnail versions (typically 274x258 — use only as last resort with Lanczos upscale)

PROCESSING STEPS (per senator):
1. Download the best available headshot from the senate.ca.gov district page
2. Determine source dimensions
3. Crop to 4:5 ratio FIRST — center crop if wider than 4:5; top crop if taller than 4:5
   (preserve eyes at ~1/3 from top; full head + shoulders visible)
4. Resize to 600x750 using Lanczos — upscale acceptable per project policy
5. Save as JPEG quality=90

UPLOAD TO SUPABASE STORAGE:
- Bucket: politician_photos
- Path: {politician_id}-headshot.jpg
- Get public URL

INSERT politician_images ROW:
```sql
INSERT INTO essentials.politician_images
  (id, politician_id, url, type, photo_license, photo_origin_url)
VALUES
  (gen_random_uuid(),
   '{politician_id}',
   '{public_url_from_storage}',
   'default',
   'public_domain',
   '{source_page_url}')  -- the senate.ca.gov district page URL
ON CONFLICT DO NOTHING;
```

photo_origin_url: Set to the senate.ca.gov district page (e.g., 'https://sd11.senate.ca.gov').
photo_license: Always 'public_domain' for official CA government website photos.

HALT POLICY:
- If a senator's page returns 503/404 for BOTH sdNN and srNN prefixes, HALT AND FLAG — do not skip.
- If a senator's page has NO photo at all (no headshot image found), HALT AND FLAG — do not skip.
- Upscale from small official photos (e.g., 274x258) is acceptable — use Lanczos.
- Do NOT use photos with superimposed text, banners, or graphics over the face (e.g., "Re-Elect" overlays).
- Do NOT use campaign website photos — official government/press pages only.

Process senators in order SD-01 through SD-40. After each successful upload + insert, proceed to the next district.
