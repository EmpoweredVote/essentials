## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ISD website → local processing → Supabase Storage | Headshot images downloaded from public ISD websites; processed locally via PIL; uploaded to Supabase Storage. Untrusted external content (image data) crosses this boundary. |
| Supabase Storage → ev-ui frontend | Uploaded images served via public Supabase Storage CDN URL embedded in `politician_images.url`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-06 | Tampering | Image content (downloaded JPEG/PNG) | mitigate | PIL re-encode forces JPEG q90 at 600x750 — strips EXIF, strips any malicious image-format payloads from original sources. Processing pipeline normalizes all inputs to a known-safe format. |
| T-88-07 | Information Disclosure | Board member photos | accept | Photos sourced from public ISD websites (D-17) — no PII beyond what is publicly published by the ISD itself. License recorded as `public_domain` consistent with established phases (Phase 87 CA pattern). |
| T-88-08 | Spoofing | Wrong photo attached to wrong politician | mitigate | Storage object key uses `{politician_id}-headshot.jpg` (UUID, not external_id) — collision-resistant. politician_images.politician_id FK ensures the row is anchored to the correct politician UUID. Task 2 human verify gate catches visible misattributions before plan completes. |
| T-88-09 | Repudiation | Live DB writes without ledger entry | mitigate | Migration 262 audit file documents every INSERT performed live with source URL + dimensions — committed to git for audit trail (same pattern as Phase 86 migration 255, Phase 87 migration 258). |
| T-88-10 | Denial of Service | Supabase Storage upload rate | accept | 35 photos max — trivial scale; no rate limiting concerns. |
| T-88-11 | Elevation of Privilege | Photo with superimposed text used for political messaging | mitigate | Per memory `feedback_headshot_no_graphics.md`, photos with "Re-Elect" banners or other superimposed graphics are rejected at sourcing time — recorded as no-photo instead. Task 1 acceptance criterion enforces this. |
</threat_model>
