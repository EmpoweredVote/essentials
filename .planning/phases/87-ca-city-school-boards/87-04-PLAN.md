<objective>
Two small, independent fixes bundled into one plan.

Gap 3 — Greyscale headshot validation: All three headshot scripts silently convert greyscale (mode='L') images to RGB before processing and upload them. This allowed 7 SFUSD B&W photos to enter Supabase Storage. The fix adds an explicit rejection guard before the crop/resize pipeline in each script. Future uploads of greyscale source photos will print a warning and skip, prompting the operator to find a color replacement.

Gap 5 — San José accent normalization: migration 257 seeded the SJUSD government_name with an accent ("San José Unified School District, California, US"). The city of San Jose uses no accent. A follow-on migration normalizes the government_name to remove the accent, making the SCHOOL section header consistent with the "San Jose" spelling used everywhere else.

Purpose: Prevents future greyscale uploads and eliminates a cosmetic inconsistency visible to all San Jose users.

Output: Three modified Python scripts + one new SQL migration file.
</objective>

In `process_image(raw_path)`, the current mode handling starts at line 115:

    if img.mode in ('RGBA', 'LA', 'P'):
        bg = Image.new('RGB', img.size, (255, 255, 255))
        if img.mode == 'P':
            img = img.convert('RGBA')
        if img.mode in ('RGBA', 'LA'):
            bg.paste(img, mask=img.split()[-1])
        img = bg
    elif img.mode != 'RGB':
        img = img.convert('RGB')

Replace the entire block with:

    if img.mode in ('L', 'LA'):
        raise ValueError(f"Greyscale image rejected (mode={img.mode}) — find a color source photo")
    if img.mode in ('RGBA', 'P'):
        bg = Image.new('RGB', img.size, (255, 255, 255))
        if img.mode == 'P':
            img = img.convert('RGBA')
        bg.paste(img, mask=img.split()[-1])
        img = bg
    elif img.mode != 'RGB':
        img = img.convert('RGB')

Note: 'LA' is removed from the RGBA/P branch because greyscale-with-alpha is now caught by the rejection guard above. RGBA and P (palette) still composite over white as before.

In all three scripts, the ValueError will propagate up to whatever loop calls the image processing function. Confirm the caller already has a try/except that logs and continues (do not add new try/except blocks — let the existing ones handle it).
