## Milestone Summary

**Key Decisions:**

- Decision: 'judicial' role_scope added to compass_topic_roles CHECK constraint (Rationale: clean separation from federal/state/local politician compass topics)
- Decision: judicial_role column on compass_topics (NULL=universal, 'judge'/'city_attorney_da' for sub-role filtering) (Rationale: single join table drives all judicial topic filtering)
- Decision: applies_judicial defaults to false (Rationale: existing cross-cutting topics must NOT appear on judicial profiles)
- Decision: Option C pivot for donor data — Legal Donor Activity without court cross-reference (Rationale: lacourt.org PAOS charges $1-$4.75/search; ~237 firms not worth manual cost for MVP)
- Decision: BarEvaluationSection omits "clean record" rows (Rationale: zero voter signal from positive State Bar/CJP status; section links to CJP UI instead)
- Decision: Plain-language description standard for judicial_disciplinary_records (Rationale: voter-facing summary of what judge did, not bureaucratic label)
- Decision: isLegalCandidate 6-condition parity across Profile.jsx + CandidateProfile.jsx (Rationale: incumbents use office_title; candidates need position_name as additional signal — intentional asymmetry)
- Decision: WHERE NOT EXISTS guard for politician inserts (Rationale: politicians table has no unique constraint on full_name)

**Issues Resolved:**

- Challenger city attorney candidates (null district_type) failed to route to JudicialCompassSection — fixed by dScope fallback (Phase 32)
