## E2E Flows

### Flow 1: Calibrated user arrives at /elections — COMPLETE ✓
1. `userAnswers.length ≥ 3` + `ev:compassMode` null → auto-enable fires → `compassMode = true`
2. `ElectionsView` renders `PoliticianCard + MiniCompass` overlay per candidate tile
3. `MiniCompass` calls `computeDisplaySpokes` → ≥3 bilateral spokes → renders `RadarChartCore` (no labels)
4. `CompassControlsBar` sticky above elections list

### Flow 2: User activates Local Lens — PARTIAL ⚠
- **Works:** Global controls bar "Local" button → `toggleLocalLens()` → all MiniCompass tiles update to `LOCAL_LENS_TOPICS` set → replacement spokes at 0.7 opacity when data thin
- **Missing:** Per-tile magnifying-glass icon entry point (MINI-05/06 gap)

### Flow 3: User deactivates Local Lens — COMPLETE ✓
`toggleLocalLens()` → `preLensSnapshot` → restores exact prior `selectedTopics` + `invertedSpokes` → `saveLocalLensState(false, null)` removes snapshot from localStorage

### Flow 4: Uncalibrated user at /elections — COMPLETE ✓
`userAnswers.length < 3` → auto-enable guard returns early → `compassMode = false` → `PoliticianCard` horizontal view; no MiniCompass; CompassControlsBar not rendered

### Flow 5: Results page dual-tab parity — COMPLETE ✓
`CompassControlsBar` at Results.jsx line 1527 (outside `activeView` ternary) → shows on both Reps tab and Elections tab simultaneously when `compassMode && (activeQuery || browseResults)`

## Open Gaps Detail

### MINI-05 + MINI-06: Per-tile Local Lens Icon

**What was specified:**
- MINI-05: Green magnifying-glass icon in top-right of each mini compass; glows green when Local Lens active, dims when inactive
- MINI-06: Clicking any tile's icon toggles the global `localLensActive` state, updating all compasses simultaneously

**What was built instead:**
- CTRL-02 (Phase 36): Local Lens toggle button in the global `CompassControlsBar` above the list

**Why deferred:** `34-CONTEXT.md` documents that "user confirmed this belongs in the Phase 36 global controls bar, not on individual mini compass tiles." However, Phase 36 implemented the global bar (CTRL-02) but did not implement the per-tile icons. Both requirements remain formally open in `REQUIREMENTS.md`.

**Disposition options:**
1. Accept the global bar as the sole Lens entry point → update REQUIREMENTS.md to mark MINI-05/06 as superseded by CTRL-02
2. Implement per-tile icon as a small v4.1 scope (30min — add a green SVG icon overlay on MiniCompass + onClick → toggleLocalLens)
